# Kindergarten Storybook MVP Design System

## 1. 设计定位

本网站是面向幼儿园老师、园所管理员和家长协作场景的个性化内容平台。界面应同时传达三件事：

- **温暖可信**：适合儿童成长、家园沟通和幼教场景，避免冰冷的企业后台感。
- **高效清晰**：老师需要快速完成建档、选案例、生成、编辑、导出，信息层级必须直接。
- **可沉淀复用**：普通绘本母本、定制绘本、共享案例和儿童档案之间的关系要被清楚表达。

设计关键词：温暖、柔和、清楚、可操作、园所级可信赖。

## 2. 设计原则

### 2.1 场景优先

导航、页面标题和卡片入口应按老师熟悉的任务组织，例如“孩子档案”“母本库”“绘本工作台”“导出分享”，不要把“模型”“Prompt”“任务队列”等技术词放在主路径。

### 2.2 主路径突出

每个页面只突出一个主要动作：

- 首页：进入登录或查看案例
- 工作台：开始新内容
- 孩子档案：新增或选择孩子
- 母本库：选择案例或派生定制绘本
- 绘本工作台：生成绘本
- 导出分享：完成交付

### 2.3 信息密度适中

老师端不是营销落地页，也不是极简阅读器。后台页面应使用清晰的卡片、列表、状态标签和分区，让重复操作可扫描。

### 2.4 儿童隐私默认可控

涉及儿童姓名、画像、兴趣、照片、分享链接、共享范围的界面，应显式展示当前状态和可见范围。共享、导出、公开案例必须有确认和脱敏提示。

## 3. 视觉令牌

当前 CSS 已在 `styles.css` 中建立基础变量。后续新增组件应优先复用这些令牌。

| `--bg` | `#f5ecd9` | 页面暖色背景 |
| `--bg-deep` | `#e9d5b5` | 背景深层、区块过渡 |
| `--panel` | `rgba(255, 250, 242, 0.86)` | 主要玻璃感面板 |
| `--panel-strong` | `#fffdf8` | 强面板、弹窗、表单底色 |

| `--muted` | `#617164` | 次级文本、说明文案 |
| `--line` | `rgba(134, 109, 64, 0.16)` | 分割线、弱边框 |
| `--brand` | `#c45b34` | 主品牌色、主按钮、重点标签 |
| `--brand-deep` | `#9e4220` | 主按钮渐变终点、强强调文本 |
| `--brand-soft` | `#f6dcc4` | 品牌浅底 |
| `--accent` | `#6f9b80` | 成功、进度、教育成长感 |

使用规则：

- 主按钮、当前导航、关键步骤激活态使用 `--brand` 到 `--brand-deep` 渐变。
- 成功、完成、可交付状态使用 `--accent` 或 `--accent-soft`。
- 页面大面积背景保持暖米色，不新增高饱和大面积色块。
- 警告或危险操作后续可新增低饱和红色，但不要复用主品牌红作为删除色。

### 3.2 阴影

| Token | 值 | 用途 |
| --- | --- | --- |
| `--shadow-lg` | `0 28px 80px rgba(102, 72, 31, 0.16)` | 页面主面板 |
| `--shadow-md` | `0 18px 42px rgba(102, 72, 31, 0.12)` | 卡片、工具栏、列表项 |

阴影应轻柔，不使用黑色硬阴影。后台密集区域可减少阴影，改用边框和浅底色区分层级。

### 3.3 字体

默认字体栈：

```css
"Avenir Next", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif
```

字号建议：

| 层级 | 桌面 | 移动端 | 用途 |
| --- | --- | --- | --- |
| Landing H1 | 56px | 34px | 首页主叙事标题 |
| Page H1 | 34px | 30px | 后台页面主标题 |
| Section H2 | 34px | 28px | 页面主要分区 |
| Card title | 20-28px | 18-24px | 卡片标题、故事页标题 |
| Body | 14-16px | 14-16px | 正文、说明 |
| Meta | 12-13px | 12-13px | 标签、状态、辅助信息 |

规则：

- 中文正文行高建议 `1.65` 到 `1.9`。
- 页面标题可适度紧凑；后台卡片内标题不要使用过大的营销标题。
- 不使用负字距。现有 landing 标题如需延续，可逐步收敛到 `letter-spacing: 0`。

### 3.4 圆角

| 圆角 | 用途 |
| --- | --- |
| `999px` | 胶囊按钮、标签、导航激活态 |
| `28-32px` | 页面主面板、首页大卡片 |
| `22-24px` | 后台业务卡片、表单分组 |
| `16-20px` | 输入框、列表项、小卡片 |
| `12-16px` | 图标底、品牌 mark |

新增后台组件优先使用 `22px` 或 `24px`，避免所有元素都做成超大圆角。

### 3.5 间距

基础间距序列：

| Token 建议 | 值 | 用途 |
| --- | --- | --- |
| `xs` | 6-8px | 标签内距、小元素间距 |
| `sm` | 10-12px | 表单项、按钮组 |
| `md` | 16-18px | 卡片内部、列表间距 |
| `lg` | 22-24px | 页面面板、主要卡片 |
| `xl` | 28-34px | Hero、页面大区块 |

规则：

- 表单、列表、卡片网格以 `12px`、`16px`、`18px` 为主。
- 页面外边距桌面使用 `24px`，移动端使用 `14px`。
- 工作台布局中左侧导航固定宽度当前为 `176px`，内容左边距当前为 `224px`。

## 4. 页面布局

### 4.1 公共 Shell

主结构：

- `.app-nav`：顶部或左侧导航。
- `.shell`：页面内容网格容器。
- `.panel`：主内容面板。
- `.preview-panel`：右侧或主内容区。
- `.form-panel`：绘本工作台左侧输入区。

桌面端后台页面：

- 左侧固定导航：`176px`
- 内容区左边距：`224px`
- 主面板内边距：`28px`

移动端：

- 导航变为顶部横向滚动。
- 所有双列或三列布局收敛为单列。
- 面板内边距降为 `18-20px`。

### 4.2 首页

首页允许更强的品牌叙事，但仍应展示真实产品对象：

- 顶部品牌导航
- Announcement
- Hero copy + Hero visual
- 功能/流程/差异化/案例 Gallery
- CTA 与联系入口

Hero 标题表达价值，不堆功能点。视觉区应展示故事卡、页面预览、生成结果或案例，而不是纯装饰图形。

### 4.3 工作台/后台页面

后台页面应使用以下结构：

1. 页面标题区：当前模块、关键状态、主要动作。
2. 关键概览区：统计、待办、最近内容或当前对象。
3. 任务区：列表、卡片、筛选、详情。
4. 状态反馈：空状态、加载、成功、错误。

避免把每个小区块都做成独立大卡片。强相关内容应在同一面板内用小卡片或分割线组织。

### 4.4 绘本工作台

绘本工作台是核心生产界面：

- 左侧输入区：只放影响当前生成结果的字段。
- 右侧预览区：展示当前故事、分页、家庭沟通文案、导出状态。
- 状态步骤：输入画像、生成绘本、导出交付。

生成链路必须有明确 loading、成功、失败和可重试状态。不要让老师猜系统是否正在处理。

## 5. 组件规范

### 5.1 按钮

按钮类型：

- `.primary`：页面主动作，例如登录、生成绘本、确认导出。
- `.secondary`：次要动作，例如查看案例、打印、返回。
- `.toolbar-button`：工具栏动作，例如保存、分享、重绘。

规则：

- 按钮最小高度 `44px`。
- 主按钮使用品牌渐变和轻阴影。
- 同一区域最多一个主按钮。
- 危险操作后续应新增 `danger` 类型，不使用 `.primary`。

### 5.2 导航

公共导航使用 `.app-nav-item`：

- 默认态：白色半透明底、绿色灰文本。
- 激活态：品牌渐变底、白色文本。
- 后台导航按业务分组：总览、母本制作、绘本定制、平台管理。

导航文案应短，优先使用业务词：控制台、母本库、我的绘本、孩子档案、绘本工作台、导出分享、共享库、园所管理、额度数据。

### 5.3 卡片

卡片类型：

- 主面板：`.panel`
- 业务卡片：`.studio-operations-card`、`.case-gallery-card`、`.child-card`
- 小列表项：`.history-item`、`.student-item`、`.delivery-item`
- 故事页卡片：`.page-card`

卡片内容顺序：

1. 状态/分类标签
2. 标题
3. 说明或元信息
4. 操作

不要在卡片内再放完整大卡片。需要分组时使用小块、分割线或浅底区域。

### 5.4 标签与状态

常用样式：

- 胶囊标签：分类、年龄、主题、共享范围。
- `output-badge`：生成、导出、完成状态。
- `summary-tag`：摘要状态。
- `toolbar-pill`：工具栏元信息。

状态语义：

| 状态 | 颜色倾向 | 示例 |
| --- | --- | --- |
| 默认 | 白底 + muted 文本 | 未开始、普通信息 |
| 激活 | 品牌浅橙 | 当前页面、当前步骤 |
| 完成 | 绿色浅底 | 已生成、已导出 |
| 处理中 | 品牌色 spinner/进度 | 正在生成、正在导出 |
| 警告 | 后续新增暖黄 | 额度不足、待审核 |
| 错误 | 后续新增柔红 | 生成失败、导出失败 |

### 5.5 表单

表单结构：

- 使用 `details.form-section-card` 分成基础设置、孩子画像、教学目标等组。
- 每组内使用 `.form-group-card`。
- 字段 label 必须清楚说明输入内容。
- 说明文案使用 `.mode-hint` 或小号 muted 文本。

输入框规范：

- 圆角 `16px`
- 内边距 `13px 14px`
- 聚焦态使用品牌浅 outline
- 长表单优先折叠次要组，主路径字段默认展开

校验规则：

- 必填字段应在 label 或错误文案中表达。
- 错误信息放在字段下方，不使用浏览器默认提示作为唯一反馈。
- 生成前校验需要说明“缺什么”和“为什么需要”。

### 5.6 列表与筛选

适用于孩子档案、最近绘本、母本库、共享库：

- 搜索框和筛选器放在列表上方。
- 列表项使用左对齐按钮式卡片，保证可点击区域足够大。
- 空状态必须给下一步动作，例如“新增孩子”“从案例生成第一本绘本”。
- 筛选后无结果要区别于初始空状态。

### 5.7 模态、Toast 与 Loading

模态：

- 用于导出选项、共享确认、危险操作确认。
- 宽度建议 `min(460px, 100%)`。
- 标题直接说明动作结果或选择目的。

Toast：

- 用于轻量结果反馈，例如“已保存草稿”“已复制分享链接”。
- 出现在右下角，最大宽度 `360px`。

Loading：

- 整本生成、导出、图片重绘需要明确 loading。
- 文案要说明当前阶段，例如“正在把孩子画像转换成课堂可用的绘本结构”。

## 6. 内容与文案

### 6.1 语气

文案应像一个专业但温和的幼教产品助手：

- 具体，不抽象。
- 少用技术词。
- 不夸大 AI 能力。
- 对儿童与家庭信息保持谨慎。

推荐：

- “当前输入会影响这一本绘本的角色、挑战和家长沟通口径。”
- “可以选择打印绘本，或切换到家长沟通版完成交付。”
- “草稿会自动保存在当前浏览器。”

避免：

- “一键颠覆幼教内容生产”
- “模型将自动理解一切”
- “公开分享孩子资料”

### 6.2 命名

统一使用以下产品词：

| 概念 | 推荐名称 |
| --- | --- |
| 普通可复用故事 | 普通绘本母本 |
| 指定儿童生成内容 | 定制绘本 |
| 儿童信息 | 孩子档案 |
| 生成与编辑区 | 绘本工作台 |
| 可复用成品 | 母本库 / 成品案例 |
| 家庭交付内容 | 导出分享 |
| 园内复用内容 | 共享库 |

## 7. 响应式规则

断点沿用当前 CSS：

- `1100px`：桌面复杂布局收敛，左侧固定导航变顶部导航。
- `720px`：移动端紧凑布局，卡片内边距和标题字号下降。

响应式要求：

- 所有网格在 `1100px` 以下应变为单列，除非内容天然可横向滚动。
- 移动端按钮组默认纵向堆叠。
- 文字不得溢出按钮、标签、卡片。
- 导航允许横向滚动，但当前激活项必须清楚可见。
- 表单输入区在移动端不 sticky。

## 8. 可访问性

最低要求：

- 所有可点击元素高度不小于 `44px`。
- 表单字段必须有可见 label。
- 颜色不能作为唯一状态表达，状态文案也要同步变化。
- 弹窗打开后应聚焦到弹窗内容，关闭后回到触发按钮。
- loading 状态需要文字说明。
- 交互按钮应支持键盘焦点态；后续可补充统一 `:focus-visible` 样式。

## 9. 动效

当前动效方向：

- 按钮 hover：轻微上移。
- 页面卡片进入：`riseIn`。
- 故事视觉元素：轻微 floating。
- loading：spinner。

规则：

- 动效只用于反馈状态和增强故事感。
- 后台列表、表单、筛选不做复杂动画。
- 动效时长建议 `160ms` 到 `420ms`。
- loading 动画必须配合文字，不能只显示转圈。

## 10. 图像与视觉资产

产品视觉应优先展示：

- 绘本封面
- 分页故事预览
- 孩子角色卡
- 家长沟通卡
- 导出/分享交付样式

现有原型使用 emoji 和渐变模拟插画。后续替换真实资产时应保持：

- 明亮、低对比、儿童友好
- 角色一致性优先于单张图惊艳程度
- 不使用过度暗色、强颗粒、恐怖或成人化风格
- 儿童照片和真实姓名在共享场景默认脱敏

## 11. Do / Don't

Do：

- 用任务语言组织页面。
- 用状态标签解释生成、导出、共享进度。
- 用温暖浅色和柔和边框建立幼教氛围。
- 让主操作清楚、靠近当前任务。
- 给空状态明确下一步。

Don't：

- 不把 AI、模型、Prompt 放在老师主路径上。
- 不在同一屏堆太多同等级大卡片。
- 不使用高饱和儿童乐园式配色。
- 不把所有内容都叫“绘本”，要区分母本、定制绘本、家园沟通内容。
- 不在共享和导出流程中省略隐私状态。

## 12. 新增页面检查清单

新增页面或组件前，检查：

- 是否复用了 `styles.css` 中已有颜色、阴影、圆角和按钮样式。
- 页面主动作是否唯一且清楚。
- 是否覆盖 loading、empty、success、error 状态。
- 是否有移动端单列布局。
- 是否避免儿童隐私信息误公开。
- 文案是否是老师能理解的业务语言。
- 是否没有引入与现有温暖幼教视觉冲突的新风格。
