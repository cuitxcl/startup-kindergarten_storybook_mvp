# 后端接口契约

## API Summary

后端一期目标是支撑当前前端 Mock 原型的完整闭环：账号登录、空间切换、园所协作、儿童档案、普通绘本、定制绘本、市场复制、投稿隐私确认、分享和导出。API 默认前缀为 `/api`，当前实现先使用内存数据，返回结构与未来数据库实现保持一致。

统一返回：

```json
{ "data": {} }
```

分页列表返回：

```json
{ "data": [], "meta": { "total": 12, "limit": 20, "offset": 0, "has_more": true } }
```

统一错误：

```json
{ "error": { "code": "not_found", "message": "资源不存在", "field": null } }
```

## Resources

- `User`：当前登录用户。
- `Workspace`：个人空间、园所空间，包含当前用户在空间内的角色。
- `WorkspaceMember`：园所成员与授权班级。
- `Classroom`：园所班级。
- `ChildProfile`：儿童档案，用于定制绘本。
- `Storybook`：普通绘本或定制绘本。
- `StorybookPage`：绘本分页正文和插图描述。
- `StorybookRole`：主角、配角、老师形象、关键道具。
- `MarketplaceTemplate`：可复制的市场模板。
- `MarketplaceSubmission`：园所投稿和隐私确认。
- `ShareLink`：绘本分享链接。
- `ExportJob`：PDF 导出任务。
- `GenerationJob`：图片生成、PDF 导出、后续 AI 文本生成等异步任务。

## Validation Rules

- 所有受保护接口必须携带 `Authorization: Bearer <token>`。
- 分页参数 `limit` 默认 50，最小 1，最大 100；`offset` 默认 0。分页列表通过 `meta.total/limit/offset/has_more` 返回总数和下一页判断。
- 配置 `KINDLEAF_AUTH_TOKEN_SECRET` 后，登录和注册会返回 `kindleaf-v1:<user_id>:<expires_at>:<signature>` 签名 token，签名使用 HMAC-SHA256，默认 7 天过期；`KINDLEAF_AUTH_TOKEN_TTL_SECONDS` 可覆盖有效期，试点 readiness 要求必须是 1 到 2592000 秒之间的整数。未配置 secret 时仍回退到第一阶段 dev token：`dev-token` 默认代表演示账号，`dev-token:<user_id>` 代表指定注册用户，用于本地开发和 smoke。后续生产化仍需补 token 刷新、撤销和登录失败限流。
- `workspace_id` 必须属于当前用户可访问空间。
- `school_teacher` 只能访问自己所在园所空间，不可管理成员、投稿审核。
- 家长资料链接和待确认队列第一版由 `school_admin` 统一管理，避免老师越过授权班级查看全园未归档儿童资料。
- `customization_plan` 生成任务必须同时指定当前空间内的普通绘本和儿童档案；`school_teacher` 还必须只能使用授权班级内儿童。
- `school_teacher` 查询生成任务列表或单个生成任务时，`GenerationJob.input_json` 会返回脱敏占位，不直接暴露原始 prompt、儿童 id 或定制参数。
- `school_admin` 可管理园所成员、班级、投稿。
- `personal_owner` 只能管理个人空间内容，不可访问园所管理接口。
- `GET /api/workspaces/:workspace_id/audit-logs` 仅返回当前 workspace 归属的审计记录。
- `GET /api/operator/audit-logs` 返回最近 100 条全局审计记录，用于查看公开分享导出、平台审核等不归属于单一 workspace 的记录。第一阶段使用 dev token mock 平台运营权限。
- 第一阶段邀请 token 直接使用 `workspace_members.id`，成员初始状态为 `invited`；接受邀请后对应 member 和 invited user 会变为 `active`；管理员可把仍处于 `invited` 的老师邀请撤回为 `revoked`，撤回后接受接口不会激活成员。后续生产化再迁移到独立 `workspace_invitations`、token hash 和更完整过期策略。
- `POST /api/parent-intakes` 不要求登录，但提交后只进入待确认队列，不直接写入正式儿童档案；如果来源资料链接绑定了班级，提交记录会继承该班级。
- `POST /api/workspaces/:workspace_id/parent-intakes/:intake_id/confirm` 只能确认当前空间下 `submitted` 状态的资料，成功后返回生成的 `ChildProfile`，并把 intake 状态改为 `confirmed`；来源提交有班级时，新儿童档案会自动进入同一班级。
- `POST /api/workspaces/:workspace_id/generation-jobs` 第一阶段只允许 `storybook_plan`、`storybook_roles`、`storybook_pages`、`storybook_page_image`、`storybook_role_reference_image`、`customization_plan` 这些任务类型。
- `POST /api/workspaces/:workspace_id/generation-jobs/:job_id/retry` 只允许重试当前空间下 `status=failed` 的任务；重试会把同一条任务从 `failed -> running -> succeeded/failed`，不新建任务，并更新 `attempt_count`、`last_error`、`next_run_at`。
- `POST /api/workspaces/:workspace_id/generation-jobs/:job_id/cancel` 只允许取消当前空间下 `status=queued/failed` 的任务；取消会写入 `status=canceled`、`output_json.schema_version=generation.canceled.v1`，并清空 `next_run_at/locked_by/locked_at`，避免被 worker 再次领取。
- `GenerationJob` 响应包含 `attempt_count`、`last_error`、`next_run_at`、`locked_by`、`locked_at`，前端可用于展示失败原因、重试次数；后续运营端可用锁字段排查卡住的生成任务。
- `GET /api/workspaces/:workspace_id/generation-provider` 与 `GET /api/operator/generation-provider` 返回统一生成能力摘要：`provider/mode/schema_version/supports_text/supports_image/real_text_ready/real_image_ready/production_ready/missing_configuration/diagnostic`。其中 `components[]` 会分别列出 `deepseek` 文本组件和 `seedream` 图片组件的 `kind/provider/configured/ready/model/endpoint/supports/required_configuration`，用于前端和运营端判断真实生成配置是否就绪；响应不返回任何密钥值。
- `GET /api/operator/storage` 仅平台运营可访问，返回 storage 摘要：`backend/exports_dir/generated_images_dir/export_max_bytes/generated_image_max_bytes/filename_validation/size_limit_enabled/download_strategy/public_direct_access`。该接口用于试点部署前确认 PDF/PNG 本地存储目录、大小上限、文件名安全规则和下载策略是否生效；不返回文件内容或密钥。
- `GET /api/operator/readiness` 仅平台运营可访问，返回 `ready/mode/checks/provider/storage`。`checks[]` 至少包含 `database/database_schema/app_host/auth_token/auth_token_ttl/generation_provider_secrets/generation_provider_config/generation_provider/storage/generation_budget/demo_seed` 十一项，用于试点上线前确认数据库可查询、核心业务表和关键字段 migration 是否完整、外部访问域名是否为非本地 HTTPS、登录 token secret 与有效期是否适合试点、DeepSeek/Seedream 密钥是否真实配置且不是明显占位值、DeepSeek/Seedream endpoint 与 model 配置是否有效、DeepSeek+Seedream 真实生成是否完整就绪、PDF/PNG 是否使用持久化目录且目录可写、生成预算上限是否配置且当前未超限、演示数据自动 seed 是否关闭；该接口只返回状态和诊断信息，不返回密钥值。
- `GenerationJob.output_json` 当前按 provider 返回统一结构，mock 使用 `schema_version=generation.mock.v1`，真实 provider 使用 `schema_version=generation.provider.v1`，并包含 `provider/mode/message`。不同任务会补充结构化业务结果：
  - `storybook_plan`：`plan.title/theme/summary/page_count/outline/role_requirements/review_points`。
  - `storybook_roles`：`roles[]` 和 `consistency_guide[]`。
  - `storybook_pages`：`pages[]`，每页包含 `page_number/title/body/illustration_prompt/status`。
  - `storybook_role_reference_image`：`image.target_id/target_type/role_id/image_url/prompt/image_mode/reference_images/style_notes`，任务成功后写回对应 `StorybookRole.reference_image_url/reference_image_prompt/reference_status=ready`。
  - `storybook_page_image`：`image.target_id/target_type/page_id/image_url/alt_text/prompt/image_mode/reference_images/edit_instruction/strength/style_notes`。
  - `customization_plan`：`customization.child_id/intensity/strategy/rewrite_points/risk_checks`。
- `storybook_roles` 和 `storybook_pages` 如果请求携带 `storybook_id`，任务成功后会自动把 `roles[]` / `pages[]` 应用到对应绘本，替换原有角色或分页；未携带 `storybook_id` 时只返回生成结果，不落库。
- 普通绘本才能派生定制绘本。
- 定制绘本必须指定 `child_id`，且儿童档案必须属于同一空间。派生时会复制母本分页和角色，再用儿童称呼、兴趣、特质和关注点做轻量改写：标题包含孩子称呼，主角替换为孩子，分页正文追加定制说明，插图提示加入孩子熟悉元素，相关页面标记为 `needs_regeneration`。
- 角色更新当前只修改已有 `StorybookRole`，支持 `name`、`role_type`、`appearance`、`story_function`、`needs_consistency`、`reference_image_url`、`reference_image_prompt`、`reference_status`；新增/删除角色留到角色生成覆盖流程。
- 页面插图任务支持 `reference_role_ids` 和 `reference_image_urls`。后端会把 `reference_role_ids` 对应角色上的 `reference_image_url` 合并进任务输入；重试时复用同一批参考图。`image_mode` 支持 `text_to_image/reference_image/edit_image`，有参考图但未指定模式时默认为 `reference_image`。
- `ExportJob` 响应包含 `id`、`storybook_id`、`status`、`file_url`、`last_error`、`created_at`、`finished_at`；PDF 生成或 storage 写入失败时会进入 `failed` 并保留截断后的 `last_error`，前端可在绘本详情页展示失败原因，后续真实 PDF 队列可复用同一查询接口轮询。
- 市场投稿必须来自普通绘本，并完成隐私确认后才能进入审核；隐私确认成功后，源普通绘本同步变为 `status=submitted`、`visibility=market_submission`。
- 平台审核通过投稿后，会创建 `marketplace_templates` 记录，写入 `source_storybook_id` 指向来源普通绘本，并把投稿状态改为 `listed`，源普通绘本同步变为 `status=listed`、`visibility=market_listed`。第一阶段平台运营接口使用 dev token mock 权限。
- 从市场复制模板时，如果模板带 `source_storybook_id`，会克隆来源绘本的分页和角色；没有来源绘本的内置平台模板使用默认分页和角色占位。

## Error Model

- `unauthorized`：未登录或 token 无效。
- `forbidden`：无权访问该空间或操作。
- `not_found`：资源不存在或不属于当前空间。
- `validation_error`：字段缺失或格式不合法。
- `state_conflict`：状态不允许当前操作。
- API smoke 会对关键 `401/403/404/400` 场景检查 `error.code` 和 `error.field`，避免前端只能依赖中文 message 判断失败类型。
- 前端 API client 会把失败响应转换为 `ApiClientError`，保留 `status/code/field/payload`，页面仍可兼容使用 `message` 展示中文提示。
