# 国内模型选型对比表

## 目标

对当前儿童定制读本产品相关的国内模型厂商做横向调研，重点评估：
- 是否适合儿童绘本插图生成
- 是否适合角色和道具一致性要求
- 是否支持参考图和多图输入
- 是否适合 MVP 到小规模生产阶段

本次调研优先关注图片模型，也会补充文本模型建议，因为本产品最终是图文一体链路。

## 产品需求背景

当前产品不是通用文生图，也不是海报设计工具，而是面向老师生成儿童定制读本。图片模型需要满足：

- 同一个孩子跨 6 到 10 页保持角色一致
- 同一个关键道具跨多页保持一致
- 同一本书画风一致
- 支持单页重绘，而不是整本重跑
- 中文场景理解稳定
- 成本适合反复试错和后续规模化

因此，本次调研的核心判断标准不是“单张图是否惊艳”，而是“是否适合稳定交付整本绘本”。

## 评估维度

本次采用以下维度做对比：

- 角色一致性潜力
- 重复道具一致性潜力
- 参考图能力
- 多图输入能力
- 图像编辑和重绘能力
- 连续组图或多轮生成能力
- 中文提示词理解
- 工程接入便利性
- 成本可控性
- 是否适合当前产品阶段

## 厂商对比结论

| 厂商 | 图片能力判断 | 适合度结论 |
|---|---|---|
| 字节火山引擎 Seedream | 最适合做当前主图片方案 | 第一推荐 |
| 腾讯混元生图 | 角色一致性和多轮编辑能力较强 | 第二推荐 |
| 阿里通义万相 | 图像编辑和平台一体化强 | 第三推荐 |
| 智谱 CogView | 更适合轻量补充，不建议做主方案 | 备选 |
| 百度 ERNIE Image | 更适合低成本试水，不建议做主方案 | 备选 |

## 1. 字节火山引擎 Seedream

### 总体判断

最适合当前产品作为主图片模型方案。

### 适合原因

- 官方明确支持 `reference_images`
- 支持单图或多图输入
- 支持图像生成、编辑、组图
- 支持连续组图能力，适合多页绘本链路
- 文档里对参考图和多图能力描述较明确

### 为什么适合儿童定制读本

这个产品最关键的是：
- 固定主角
- 固定关键道具
- 分页生成
- 单页修复

Seedream 的参考图和多图输入能力，天然更适合构建：
- `character_profile`
- `prop_profile`
- 参考图驱动的多页插图生成

### 风险和注意点

- 最终一致性效果仍需要你自己做 A/B 测试
- 如果批量任务增加，需要验证并发和限流策略
- 需要尽早设计 Prompt 模板和参考图链路，否则模型优势无法转化为稳定交付

### 官方资料

- 火山引擎图片生成与编辑文档：
  - <https://www.volcengine.com/docs/6492/2221472>
- Seedream API 参考：
  - <https://www.volcengine.com/docs/82379/1541523>

## 2. 腾讯混元生图

### 总体判断

是非常稳的第二选择，也适合做备份链路或对照实验。

### 适合原因

- 官方明确支持文生图、单图生图、多图生图
- 官方表述中直接提到角色和画面的一致性稳定输出
- 支持多轮对话式修改图片
- 中文理解能力较强

### 为什么适合儿童定制读本

腾讯混元的优势在于：
- 中文语义理解自然
- 多轮修改体验较适合老师反复调整
- 做单页修改和编辑链路比较顺

### 风险和注意点

- 默认并发能力在文档里有明确限制
- 如果要做班级批量生成，后期可能会遇到吞吐瓶颈
- 需要尽早关注成本与任务队列设计

### 官方资料

- 产品页：
  - <https://cloud.tencent.com/product/aiart>
- 多轮生图接口：
  - <https://cloud.tencent.com/document/api/1729/110428>
- 计费和并发说明：
  - <https://cloud.tencent.com/document/product/1668/90896>

## 3. 阿里通义万相

### 总体判断

不是当前图片稳定性主战场的最优，但非常适合做一体化平台方案。

### 适合原因

- 图像编辑能力明确
- 支持参考卡通形象生成
- 和阿里百炼文本模型结合度高
- 后端接入、鉴权、平台管理比较完整

### 为什么适合儿童定制读本

如果未来你想把下面这些都统一放在一个平台：
- 故事生成
- 角色卡生成
- 图像生成
- 图像编辑
- 企业级后端服务

阿里是很顺的一套栈。

### 风险和注意点

- 官方公开资料里，对跨页角色一致性的强调不如字节和腾讯直接
- 更适合作为整套系统平台，而不一定是当前最强的绘本一致性方案

### 官方资料

- 万相图像编辑：
  - <https://help.aliyun.com/zh/model-studio/wanx-image-edit-api-reference>
- 百炼文本模型：
  - <https://help.aliyun.com/zh/model-studio/text-generation-model/>
- 百炼模型价格：
  - <https://help.aliyun.com/zh/model-studio/model-pricing>

## 4. 智谱 CogView

### 总体判断

不建议作为当前项目的主图片方案，但可以作为轻量补充能力。

### 适合原因

- 中文提示词能力好
- 对汉字内容支持较好
- 单次调用成本较低

### 更适合的用途

- 封面图
- 活动海报
- 带中文标题的宣传图
- 低成本试验图

### 不适合作为主方案的原因

- 当前公开资料里，对多参考图一致性和连续组图能力强调不强
- 不如字节和腾讯贴近“多页一致性绘本”场景

### 官方资料

- CogView-4 文档：
  - <https://docs.bigmodel.cn/cn/guide/models/image-generation/cogview-4>

## 5. 百度 ERNIE Image

### 总体判断

更适合低成本试水，不建议做当前项目的主图片方案。

### 适合原因

- 价格较低
- 接口简单
- 可以作为快速验证通用文生图能力的入口

### 不适合作为主方案的原因

- 官方资料里，对角色一致性、多图输入、跨页连续性支持不够突出
- 不足以支撑你当前产品最核心的稳定性要求

### 官方资料

- 百度千帆图像接口：
  - <https://cloud.baidu.com/doc/qianfan-api/s/Imo9g5a6a>
- 计费：
  - <https://cloud.baidu.com/doc/qianfan/s/wmh4sv6ya>

## 文本模型补充建议

虽然这次核心在图片，但当前产品不是纯图片产品，仍需要文本模型来做：
- 故事生成
- 角色卡规范化
- 道具卡规范化
- 场景说明生成
- Prompt 结构化输出

### 推荐优先级

| 厂商 | 文本建议 |
|---|---|
| 阿里 Qwen | 第一推荐 |
| DeepSeek | 第二推荐 |
| 腾讯混元 | 可选 |
| Kimi | 可选 |

### 判断原因

`Qwen` 更适合当前产品的原因：
- 官方强调工具调用、长上下文、结构化任务能力
- 在中文任务、JSON 输出、角色卡整理上更适合做中枢模型

官方资料：
- <https://help.aliyun.com/zh/model-studio/text-generation-model/>

`DeepSeek` 更适合：
- 较强中文推理
- 较强成本效率比

官方资料：
- <https://platform.deepseek.com/api-docs/>

## 适配当前产品阶段的最终排序

### 图片模型排序

1. 字节 Seedream
2. 腾讯混元生图
3. 阿里通义万相
4. 智谱 CogView
5. 百度 ERNIE Image

### 文本模型排序

1. 阿里 Qwen
2. DeepSeek
3. 腾讯混元文本
4. Kimi

## 推荐组合

### 组合 A：当前最推荐

- 文本：阿里 Qwen
- 图片：字节 Seedream

原因：
- 文本结构化和角色卡生成交给 Qwen
- 图片一致性主链路交给 Seedream
- 最符合当前产品以图片稳定性为核心的阶段目标

### 组合 B：全腾讯方案

- 文本：腾讯混元
- 图片：腾讯混元生图

原因：
- 一家平台统一接入
- 中文语义和对话式编辑链路顺

### 组合 C：全阿里方案

- 文本：Qwen
- 图片：通义万相

原因：
- 平台整合度高
- 后续企业化和工程化最顺

## 最终结论

如果只问“国内哪个厂商最适合当前这个产品”，答案是：

- 图片主方案：字节
- 企业级稳妥备选：腾讯
- 一体化平台方案：阿里

如果问“当前最建议怎么接”，答案是：

- 文本用 `Qwen`
- 图片用 `Seedream`

这套组合最符合当前产品以“儿童绘本插图稳定性”为核心目标的实际需要。
