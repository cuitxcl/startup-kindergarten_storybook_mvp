# 产品功能总纲

## 1. 产品定位

本产品是一套面向个人用户和园所用户的儿童绘本生成系统。

系统的核心能力是：

- 生成不绑定具体孩子的普通绘本。
- 基于普通绘本和孩子资料派生定制绘本。
- 通过账号、空间和空间角色区分个人创作、园所协作和外部链接访问。
- 通过绘本市场提供高质量起点，降低用户从空白开始创作的成本。

第一版不把产品做成纯幼儿园工具，也不做成泛化 AI 创作平台，而是围绕“儿童绘本生成”建立一套可复用、可扩展的内容生产流程。

## 2. 核心问题

目标用户面临的问题：

- 从 0 写一本适合儿童阅读的绘本门槛高。
- 老师和家长很难稳定产出有教学目标、结构完整、语言合适的绘本。
- 如果要给某个孩子做专属绘本，从头生成成本高，质量也不稳定。
- 园所希望沉淀优秀绘本内容，但普通内容和含儿童信息的定制内容需要明确隔离。
- 用户需要可复用的优质模板，而不是每次都面对空白输入框。

产品的解决思路：

```text
普通绘本生成
-> 保存为可复用内容
-> 基于普通绘本派生定制绘本
-> 通过市场和园所内容沉淀复用优秀作品
```

## 3. 目标用户

### 3.1 个人用户

个人用户可以是家长、普通创作者，也可以是老师的个人身份。

核心需求：

- 生成普通儿童绘本。
- 管理自己的孩子资料。
- 为自己的孩子生成定制绘本。
- 导出 PDF 或后续分享给家人。

### 3.2 园所老师

园所老师在园所空间中使用系统。

核心需求：

- 为课堂、规则引导、家园沟通生成普通绘本。
- 管理或使用班级儿童档案。
- 为班级里的单个孩子生成定制绘本。
- 导出或分享给家长。
- 复用园所已有优秀普通绘本。

### 3.3 园所管理员

园所管理员负责园所空间的基础管理。

核心需求：

- 邀请老师加入园所空间。
- 管理班级。
- 查看园所内容和基础用量。
- 推动园所优秀普通绘本沉淀和投稿到绘本市场。

### 3.4 链接访问者

链接访问者不一定注册登录。

核心需求：

- 通过资料填写链接提交孩子资料。
- 通过分享链接查看或下载绘本。

### 3.5 平台运营

平台运营主要负责后续平台市场和模板治理。

核心需求：

- 管理平台模板。
- 审核园所投稿作品。
- 上架、下架市场内容。
- 后续查看平台运营数据。

第一版可以预留平台运营角色，但不把完整后台作为主流程。

## 4. 基础模型：账号、空间、空间角色

系统不把用户固定写死为普通用户或老师，而是采用：

```text
User 账号
  └── Workspace 空间
        └── Workspace Role 空间角色
```

### 4.1 账号 User

账号代表一个可登录的人。

规则：

- 每个注册账号默认拥有一个个人空间。
- 一个账号可以加入一个或多个园所空间。
- 同一个账号可以在个人空间中创作个人绘本，也可以在园所空间中作为老师工作。
- 账号本身不直接决定全部权限，权限由当前空间和空间角色决定。

### 4.2 空间 Workspace

空间是数据归属和权限边界。

第一版空间类型：

| 空间类型 | 说明 |
| --- | --- |
| 个人空间 | 每个注册账号默认拥有，用于个人绘本、个人孩子资料、个人导出 |
| 园所空间 | 园所/机构协作空间，用于老师、班级儿童、园所绘本和管理 |
| 链接访问 | 不是完整空间，用于非登录用户提交资料或查看分享内容 |

所有核心数据都应归属到某个空间：

- 普通绘本
- 定制绘本
- 孩子资料
- 图片资产
- 导出记录
- 分享链接
- 后续额度和用量记录

### 4.3 空间角色 Workspace Role

空间角色只在当前空间内生效。

第一版角色：

| 角色 | 生效范围 | 核心定位 |
| --- | --- | --- |
| `personal_owner` | 个人空间 | 个人创作和个人孩子定制 |
| `school_teacher` | 园所空间 | 老师生成绘本、使用班级儿童档案、交付家长 |
| `school_admin` | 园所空间 | 管理园所成员、班级、内容和基础用量 |
| `link_visitor` | 链接访问 | 非登录或轻登录参与者，提交资料或查看绘本 |
| `platform_operator` | 平台后台 | 平台模板、市场和审核管理，后续完善 |

## 5. 空间角色功能

### 5.1 personal_owner

`personal_owner` 是个人空间拥有者。

可以做：

| 功能域 | 第一版功能 | 延后功能 |
| --- | --- | --- |
| 账号与空间 | 注册/登录，自动创建个人空间 | 多个人空间、空间迁移 |
| 普通绘本 | 创建、编辑、复制、删除、导出自己的普通绘本 | 发布到平台市场 |
| 孩子资料 | 创建、编辑、删除自己的孩子资料 | 家庭成员协作、资料授权 |
| 定制绘本 | 基于自己的普通绘本和孩子资料生成定制绘本 | 基于平台付费模板生成 |
| 资产与导出 | 查看导出记录、下载 PDF | 高清导出包、云端相册 |
| 分享 | 第一版可先不做，或仅下载 PDF | 分享链接、亲友查看 |
| 额度/支付 | 第一版可先不做 | 个人额度、购买套餐 |

不可以做：

- 查看园所空间内容。
- 查看班级儿童档案。
- 使用园所额度。
- 管理老师、班级、园所内容。
- 进入园所内部共享内容。
- 修改平台模板。

主路径：

```text
登录
-> 进入个人空间
-> 创建普通绘本
-> 编辑
-> 导出 PDF
```

```text
登录
-> 创建孩子资料
-> 选择普通绘本
-> 生成定制方案
-> 确认
-> 生成定制绘本
-> 编辑
-> 导出 PDF
```

### 5.2 school_teacher

`school_teacher` 是某个账号在某个园所空间里的老师角色。

可以做：

| 功能域 | 第一版功能 | 延后功能 |
| --- | --- | --- |
| 空间 | 进入园所空间，查看当前园所和授权班级 | 多园所切换 |
| 普通绘本 | 创建、编辑、导出自己创建的园所普通绘本 | 发布为园所共享母本 |
| 班级儿童 | 查看、新增、编辑自己班级儿童档案 | 跨班级授权、批量导入 |
| 定制绘本 | 基于园所普通绘本和班级儿童生成定制绘本 | 全班批量定制 |
| 家长资料 | 创建资料提交链接，确认家长提交内容 | 家长账号绑定 |
| 导出交付 | 导出 PDF | 家长分享链接、二维码 |
| 园所内容复用 | 使用已发布的园所普通绘本 | 园所共享库、复制改编 |
| 额度 | 使用园所额度，第一版前端可弱化 | 额度明细、成本提示 |

不可以做：

- 查看其他园所内容。
- 查看未授权班级儿童档案。
- 管理老师账号。
- 修改园所额度配置。
- 查看复杂运营报表。
- 管理平台市场和平台审核。

主路径：

```text
进入园所空间
-> 创建普通绘本
-> 保存为园所内容
-> 导出 PDF
```

```text
进入园所空间
-> 选择园所普通绘本
-> 选择班级儿童
-> 检查儿童档案
-> 生成定制方案
-> 确认
-> 生成定制绘本
-> 导出或分享给家长
```

### 5.3 school_admin

`school_admin` 是园所空间管理员。

第一版应保持轻量，不抢老师主流程。

可以做：

| 功能域 | 第一版功能 | 延后功能 |
| --- | --- | --- |
| 园所设置 | 查看/编辑园所基础信息 | 品牌配置、合同信息 |
| 成员管理 | 邀请老师、停用老师、查看成员 | 角色细分、审批流 |
| 班级管理 | 创建、编辑、归档班级 | 年级结构、升班 |
| 儿童档案 | 查看园所儿童概览 | 跨班级批量管理 |
| 园所内容 | 查看园所普通绘本和定制绘本 | 审核发布、精选内容库 |
| 额度/用量 | 查看基础生成量、导出量、额度余额 | 成本报表、套餐管理 |
| 市场投稿 | 提交园所优秀普通绘本到市场审核 | 投稿统计、作者展示 |
| 权限 | 管理老师是否可访问某班级 | 细粒度权限矩阵 |

不做或延后：

- 平台级模板审核。
- 平台公开内容管理。
- 家长支付管理。
- 复杂 BI 报表。
- 自动内容风控。

主路径：

```text
进入园所空间管理视图
-> 邀请老师
-> 创建班级
-> 查看园所绘本和基础用量
```

### 5.4 link_visitor

`link_visitor` 是通过链接访问系统的人，不一定登录。

场景：

| 场景 | 第一版功能 | 延后功能 |
| --- | --- | --- |
| 家长资料提交 | 打开老师分享的资料填写链接，提交孩子资料 | 修改提交、进度查看 |
| 绘本查看 | 打开分享链接查看或下载绘本 | 评论、收藏、转发 |
| 隐私确认 | 看到资料用途说明和分享范围 | 授权管理 |
| 登录转化 | 可不登录完成一次性动作 | 注册成为 `personal_owner` |

限制：

- 只能访问链接授权范围内的数据。
- 不能浏览老师其他内容。
- 不能查看园所儿童档案。
- 不能编辑绘本。
- 不能进入园所空间。

### 5.5 platform_operator

`platform_operator` 是平台后台角色。

第一版可以预留，后续再完善。

建议功能：

- 平台模板管理。
- 绘本市场内容审核。
- 园所投稿审核。
- 上架、下架市场模板。
- 平台运营数据查看。
- 内容安全与风控。
- 模型成本监控。

## 6. 用户成为 school_teacher 的方式

第一版只采用“园所管理员邀请老师”路径。

流程：

```text
school_admin
-> 成员管理
-> 邀请老师
-> 输入老师邮箱或手机号
-> 可选：指定可访问班级
-> 系统生成邀请链接或邀请码
-> 老师打开邀请链接
-> 注册或登录
-> 查看邀请信息
-> 接受邀请
-> 成为该园所空间的 school_teacher
```

邀请信息应展示：

- 园所名称。
- 邀请人。
- 将获得的角色：老师。
- 可访问班级，如果有。
- 接受后将进入哪个园所空间。

邀请状态：

| 状态 | 含义 |
| --- | --- |
| `pending` | 已创建，未接受 |
| `accepted` | 已接受 |
| `expired` | 已过期 |
| `revoked` | 已撤回 |

接受邀请后创建空间成员记录：

```text
workspace_member
- user_id
- workspace_id
- role: school_teacher
- status: active
- invited_by
- invited_contact
- classroom_ids，可选
- joined_at
```

第一版不做：

- 老师搜索园所并申请加入。
- 用户自己把自己设置成老师。
- 复杂入职审批流。

## 7. 普通绘本

### 7.1 定义

普通绘本是不绑定具体孩子、可供个人阅读、全班共读或老师教学使用的完整图文绘本。

普通绘本也是后续定制绘本的来源。

可以理解为：

```text
普通绘本 = 可直接使用的完整绘本 + 可复用母本
```

### 7.2 核心流程

```text
1. 填写绘本需求
2. 生成绘本方案
3. 用户确认/调整方案
4. 生成角色与关键道具设定
5. 用户确认/修改角色
6. 生成分页图文内容
7. 用户编辑分页内容
8. 生成插图
9. 预览、保存、导出
```

### 7.3 关键原则

- 先确认绘本方案，再确认角色设定，最后生成完整绘本。
- 绘本方案负责确认“故事讲什么”。
- 角色设定负责确认“谁来演这个故事”。
- 角色设定不早于故事方案确认，避免故事调整后角色返工。
- 普通绘本保存后可作为定制绘本来源。

### 7.4 绘本方案内容

绘本方案应包含：

- 绘本标题。
- 一句话故事简介。
- 教学目标或阅读目标。
- 情绪/行为目标。
- 页数结构。
- 角色需求。
- 关键道具/场景。
- 画面基调。

这一阶段只生成“角色需求”，不生成详细外观。

### 7.5 角色与道具设定

角色设定包含：

- 角色名。
- 角色类型。
- 外观描述。
- 服装或代表颜色。
- 性格。
- 故事作用。
- 是否重复出现。
- 是否需要插图一致性。

关键道具设定包含：

- 道具名。
- 外观描述。
- 故事作用。
- 是否重复出现。

## 8. 定制绘本

### 8.1 定义

定制绘本是基于一份已完成的普通绘本，为某个孩子生成的专属版本。

第一版不从零生成定制绘本，而是从普通绘本派生。

```text
普通绘本
-> 选择孩子
-> 生成定制方案
-> 确认后生成独立定制副本
```

### 8.2 核心流程

```text
1. 选择普通绘本
2. 选择孩子
3. 检查儿童档案
4. 选择定制强度
5. 生成定制方案
6. 用户确认/调整定制方案
7. 生成独立定制绘本副本
8. 用户编辑副本
9. 生成或重绘需要变化的插图
10. 预览、导出 PDF 或分享
```

### 8.3 关键原则

- 定制绘本优先从普通绘本派生。
- 原普通绘本永远不被覆盖。
- 用户确认的是“这本书会怎样变成某个孩子的版本”，不是底层角色映射。
- 系统先生成可审核的定制方案，再生成完整定制副本。
- 儿童档案不足时要提示补充。
- 个性化强度要可控，方便后续扩展到批量定制。
- 隐私提示必须进入流程。

### 8.4 儿童档案检查

必需信息：

| 字段 | 用途 |
| --- | --- |
| 孩子称呼/昵称 | 用于标题、正文和主角称呼 |
| 年龄段或班级年龄 | 控制语言难度和故事表达 |
| 至少一个个性化元素 | 避免只是名字替换 |

建议补充信息：

- 性格特点。
- 老师观察。
- 教学关注点。
- 喜欢的颜色。
- 外观或角色偏好。
- 家庭共读寄语。

规则：

- 缺少必需信息时，阻止生成并提示补充。
- 缺少建议信息时，允许继续，但提示个性化程度会较弱。

### 8.5 定制强度

第一版只保留两个强度：

| 模式 | 说明 | 适合场景 |
| --- | --- | --- |
| 快速定制 | 替换称呼、标题、关键道具，尽量保留原文和原图 | 批量生成、低成本交付 |
| 标准定制 | 改写关键页面，并重绘关键插图 | 单个孩子常规定制、家园沟通 |

深度定制延后。

### 8.6 定制方案内容

定制方案应包含：

- 定制标题。
- 主角处理。
- 保留角色。
- 个性化元素。
- 改写页面。
- 重绘插图。
- 隐私提示。
- 预计消耗。

定制方案需要明确：

- 会改什么。
- 会保留什么。
- 哪些页面需要重新生成文字。
- 哪些页面需要重新生成插图。
- 使用了哪些儿童信息。

## 9. 孩子资料

孩子资料是定制绘本的输入。

### 9.1 个人空间孩子资料

个人空间中建议称为：

- 我的孩子。
- 孩子资料。
- 定制对象。

第一版字段：

| 字段 | 要求 |
| --- | --- |
| 孩子称呼/昵称 | 必填 |
| 年龄段 | 必填 |
| 兴趣或特点 | 至少一项 |
| 性格特点 | 可选 |
| 喜欢的颜色 | 可选 |
| 喜欢的玩具/活动 | 可选 |
| 最近想引导的行为 | 可选 |
| 外观偏好/角色偏好 | 可选 |
| 家庭寄语 | 可选 |

### 9.2 园所空间儿童档案

园所空间中建议称为：

- 儿童档案。
- 班级儿童。

第一版字段：

- 孩子称呼/昵称。
- 年龄段。
- 班级。
- 兴趣标签。
- 性格特点。
- 老师观察。
- 教学关注点。
- 喜欢的颜色、玩具或活动。
- 外观或角色偏好。
- 家长提交来源。

### 9.3 家长资料提交

园所老师可以创建资料填写链接。

流程：

```text
老师创建家长资料填写链接
-> 家长通过链接填写资料
-> 系统生成待确认提交
-> 老师确认
-> 写入儿童档案
```

家长第一版不需要登录。

## 10. 绘本市场

### 10.1 市场定位

绘本市场第一版不是交易市场，而是：

> 平台精选和园所投稿的可复用绘本模板库。

市场用于降低用户从空白开始创作的成本，为普通绘本和定制绘本提供高质量起点。

第一版不做：

- 作者入驻。
- 付费购买。
- 分成结算。
- 评论评分社区。
- 复杂版权协议。
- 用户自由发布市场内容。

### 10.2 市场内容类型

市场内容分为：

```text
绘本市场
├── 平台模板
└── 园所投稿作品
```

| 内容类型 | 是否进入市场 | 规则 |
| --- | --- | --- |
| 平台官方模板 | 可以 | 平台直接上架 |
| 平台精选普通绘本 | 可以 | 平台维护 |
| 园所普通绘本 | 可以 | 投稿、脱敏确认、平台审核后上架 |
| 定制绘本 | 默认不可以 | 需先转成脱敏模板，第一版不做 |
| 含儿童照片绘本 | 不可以 | 第一版禁止 |
| 老师私有草稿 | 不可以 | 不应公开 |
| 园所内部绘本 | 不自动进入市场 | 需主动投稿 |

### 10.3 市场使用流程

个人用户：

```text
进入绘本市场
-> 按主题筛选
-> 查看模板详情
-> 复制到个人空间
-> 编辑为自己的普通绘本
-> 可选：选择孩子生成定制绘本
```

老师：

```text
进入绘本市场
-> 选择教学主题模板
-> 复制到园所空间
-> 编辑或导出普通绘本
-> 可选：选择班级孩子生成定制绘本
```

市场模板不会被用户直接修改。

```text
市场模板
-> 复制到当前空间
-> 成为该空间里的普通绘本副本
```

### 10.4 市场模板详情

模板详情应包含：

- 标题。
- 简介。
- 适用年龄。
- 使用场景。
- 教学目标或阅读目标。
- 页数结构。
- 角色需求。
- 示例页面。
- 是否支持定制。
- 来源：平台模板或园所投稿。

### 10.5 园所投稿市场

园所可以将优秀普通绘本投稿到市场。

第一版规则：

- 只允许普通绘本投稿。
- 定制绘本默认不可投稿。
- 含儿童照片或具体儿童隐私的内容不可投稿。
- 投稿后需要平台审核。
- 审核通过后作为市场模板公开展示和复用。

推荐流程：

```text
school_admin 选择园所优秀普通绘本
-> 点击投稿到绘本市场
-> 系统生成投稿预览
-> 确认不含儿童隐私
-> 提交平台审核
-> 平台审核通过
-> 上架市场
```

后续可支持：

```text
school_teacher 推荐作品
-> school_admin 审批
-> 平台审核
-> 上架市场
```

投稿状态：

| 状态 | 含义 |
| --- | --- |
| `draft` | 投稿草稿 |
| `submitted` | 已提交平台审核 |
| `approved` | 审核通过 |
| `rejected` | 审核拒绝 |
| `listed` | 已上架市场 |
| `unlisted` | 已下架 |

### 10.6 市场来源展示

市场可以展示来源，但要保护园所和老师隐私。

可选策略：

- 显示“平台精选”。
- 显示“园所投稿”。
- 显示园所名称，不显示老师姓名。
- 匿名展示园所来源。

第一版建议：

```text
默认显示：园所投稿
可选显示：园所名称
默认不显示具体老师姓名
```

## 11. 数据归属规则

### 11.1 绘本归属

| 来源 | 归属 |
| --- | --- |
| 个人空间创建 | 个人空间 |
| 园所空间创建 | 园所空间 |
| 市场模板复制到个人空间 | 个人空间 |
| 市场模板复制到园所空间 | 园所空间 |
| 定制绘本副本 | 当前空间 |

### 11.2 孩子资料归属

| 类型 | 归属 |
| --- | --- |
| 个人孩子资料 | 个人空间 |
| 班级儿童档案 | 园所空间 |
| 家长提交资料 | 提交链接对应的园所空间，老师确认后写入儿童档案 |

### 11.3 市场内容归属

市场内容是平台级资源。

用户使用市场内容时，必须复制到自己的空间，不能直接修改市场原件。

```text
marketplace_template
-> copy/use
-> workspace_storybook
```

## 12. 第一版范围

### 12.1 P0

P0 聚焦个人空间和绘本生成基本闭环。

- 注册/登录。
- 每个账号默认拥有个人空间。
- `personal_owner`。
- 创建普通绘本。
- 普通绘本方案确认。
- 角色与道具确认。
- 生成分页图文内容。
- 编辑普通绘本。
- 保存普通绘本。
- PDF 导出。
- 创建孩子资料。
- 从普通绘本生成定制绘本。
- 定制方案确认。
- 生成定制副本。
- 编辑定制绘本。
- 定制绘本 PDF 导出。

### 12.2 P1

P1 引入园所老师场景。

- 园所空间。
- `school_admin` 邀请老师。
- 用户接受邀请成为 `school_teacher`。
- 老师进入园所空间。
- 班级管理基础能力。
- 班级儿童档案。
- 老师在园所空间创建普通绘本。
- 老师基于班级儿童生成定制绘本。
- 家长资料提交链接。
- 老师确认家长提交资料。

### 12.3 P2

P2 引入园所管理和市场复用。

- `school_admin` 基础管理视图。
- 成员管理。
- 班级管理增强。
- 园所内容概览。
- 基础额度/用量。
- 绘本市场浏览。
- 平台模板复制到个人空间或园所空间。
- 园所普通绘本投稿市场。
- 平台审核后市场上架。

### 12.4 P3

P3 做高级能力。

- 全班批量定制。
- 家长账号。
- 支付和套餐。
- 平台运营后台。
- 平台模板审核体系。
- 复杂市场交易。
- 作者分成。
- 评论评分。
- 自动脱敏。
- 内容安全审核。

## 13. 非目标

第一版不做：

- 完整交易市场。
- 家长账号体系。
- 支付购买和分成结算。
- 平台公开社区。
- 复杂园所权限矩阵。
- 跨园所协作。
- 定制绘本公开投稿。
- 含儿童照片内容上架市场。
- 深度定制模式。
- 全班批量生成。

## 14. 验收标准

### 14.1 普通绘本

- 用户能从个人空间创建一本普通绘本。
- 系统能先生成绘本方案，而不是直接生成完整绘本。
- 用户能确认或修改绘本方案。
- 系统能基于确认方案生成角色与关键道具设定。
- 用户能确认或修改角色设定。
- 系统能生成分页图文内容。
- 用户能编辑分页内容。
- 用户能保存并导出 PDF。

### 14.2 定制绘本

- 用户能创建孩子资料。
- 用户能从已有普通绘本发起定制。
- 系统能检查孩子资料是否满足定制要求。
- 系统能生成定制方案。
- 用户能确认或调整定制方案。
- 系统能生成独立定制绘本副本。
- 修改定制副本不会影响原普通绘本。
- 用户能导出定制绘本 PDF。

### 14.3 空间与角色

- 注册账号自动拥有个人空间。
- 用户只能访问个人空间内的数据。
- `school_admin` 能邀请老师。
- 被邀请用户接受邀请后，在对应园所空间获得 `school_teacher` 角色。
- 园所空间内容和个人空间内容互相隔离。

### 14.4 绘本市场

- 用户能浏览市场模板。
- 用户能查看模板详情。
- 用户能把市场模板复制到自己的当前空间。
- 复制后的内容可以编辑，不影响市场原模板。
- 园所普通绘本可以提交市场审核。
- 未审核通过的园所作品不会出现在市场。

## 15. 风险与假设

### 15.1 风险

- 如果个人空间和园所空间在界面上表达不清，用户可能不知道当前内容归属哪里。
- 如果市场过早做成交易系统，会拖慢普通绘本和定制绘本主流程。
- 如果定制绘本直接从零生成，会导致成本高、质量不稳定、难以批量化。
- 如果允许定制绘本直接公开投稿，会带来儿童隐私风险。
- 如果园所管理员后台过早复杂化，会抢占老师主流程资源。

### 15.2 假设

- 第一版先以 PDF 导出作为主要交付形式。
- 真实图片生成可以先弱化，但流程要保留插图描述、插图生成和重绘的扩展点。
- 每个用户默认拥有个人空间。
- 园所老师身份必须通过园所管理员邀请获得。
- 绘本市场第一版是模板发现和复用，不是付费交易。
- 园所投稿市场第一版只允许普通绘本，不允许含儿童隐私的定制绘本。
