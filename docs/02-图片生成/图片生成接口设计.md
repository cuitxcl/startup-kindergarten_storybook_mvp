# 图片生成接口设计

## 目标

定义儿童定制读本项目中的图片生成接口、任务状态、重试机制和成本记录方式，支持：
- 多步生成链路
- 单页独立重试
- 稳定性分析
- 成本追踪

这份文档面向后端和前端联调。

## 设计原则

- 图片生成必须异步化
- 角色生成和分页生成分离
- 单页任务和整本任务分离
- 所有生成任务必须可追踪、可重试、可计费

## 核心对象

### 1. Character Profile

结构见 [图片生成Prompt规范.md](/Users/Zhuanz/RustroverProjects/kindergarten_storybook_mvp/docs/02-图片生成/图片生成Prompt规范.md)。

### 2. Scene Spec

表示单页图片的场景输入。

### 3. Style Spec

表示固定风格配置。

### 4. Image Asset

表示一次生成后的图片资产。

### 5. Generation Task

表示一次异步图片生成任务。

## 推荐流程

### 流程 1：生成标准角色图

1. 前端提交儿童档案
2. 后端创建 `character_profile`
3. 后端发起标准角色图生成任务
4. 返回任务 ID
5. 前端轮询任务状态
6. 成功后拿到 `reference_image_id`

### 流程 2：生成单页图片

1. 前端提交 `character_profile_id`、`scene_spec`、`style_id`
2. 后端创建单页图片任务
3. 后端调用模型生成图片
4. 返回任务状态和图片资产信息

### 流程 3：生成整本读本图片

1. 前端提交故事 ID
2. 后端读取角色卡、风格和所有分页场景
3. 后端按页创建子任务
4. 每页独立生成
5. 汇总整本状态

## 接口列表

## 1. 创建角色卡

### `POST /api/characters`

#### 请求体

```json
{
  "child_id": "child_001",
  "name": "小雨",
  "nickname": "雨雨",
  "age_group": "5岁",
  "gender_expression": "女孩",
  "raw_traits": {
    "hair": "黑色短发",
    "outfit": "经常穿黄色衣服",
    "accessory": "兔子发夹",
    "interests": ["积木", "小兔子"]
  },
  "photo_asset_id": "asset_photo_001"
}
```

#### 响应体

```json
{
  "character_profile_id": "char_001",
  "character_profile_version": 1,
  "status": "created"
}
```

## 2. 获取角色卡

### `GET /api/characters/:character_profile_id`

#### 响应体

```json
{
  "character_profile_id": "char_001",
  "character_profile_version": 1,
  "data": {
    "name": "小雨",
    "age_group": "5岁",
    "hair": "黑色短发，齐刘海",
    "outfit_top": "黄色卫衣",
    "outfit_bottom": "蓝色背带裤",
    "accessory": "小兔子发夹"
  }
}
```

## 3. 生成标准角色图

### `POST /api/images/reference`

#### 请求体

```json
{
  "character_profile_id": "char_001",
  "style_id": "storybook_flat_v1"
}
```

#### 响应体

```json
{
  "task_id": "task_ref_001",
  "task_type": "reference_image_generation",
  "status": "queued"
}
```

## 4. 生成单页图片

### `POST /api/images/pages`

#### 请求体

```json
{
  "storybook_id": "book_001",
  "page_id": "page_003",
  "character_profile_id": "char_001",
  "character_profile_version": 1,
  "reference_image_id": "img_ref_001",
  "style_id": "storybook_flat_v1",
  "scene_spec": {
    "page_number": 3,
    "location": "幼儿园教室",
    "action": "把积木递给同学",
    "emotion": "开心，主动分享",
    "composition": "中景，主角位于画面中央偏左",
    "camera_angle": "平视",
    "background_elements": ["玩具柜", "地垫", "少量积木"],
    "must_show": ["主角主动递出积木"],
    "must_avoid": ["背景拥挤"]
  }
}
```

#### 响应体

```json
{
  "task_id": "task_page_003_v1",
  "task_type": "page_image_generation",
  "status": "queued"
}
```

## 5. 批量生成整本图片

### `POST /api/images/storybooks`

#### 请求体

```json
{
  "storybook_id": "book_001",
  "character_profile_id": "char_001",
  "character_profile_version": 1,
  "reference_image_id": "img_ref_001",
  "style_id": "storybook_flat_v1",
  "page_ids": ["page_001", "page_002", "page_003", "page_004", "page_005", "page_006"]
}
```

#### 响应体

```json
{
  "task_id": "task_book_001_img_batch_01",
  "task_type": "storybook_image_generation",
  "status": "queued",
  "page_task_count": 6
}
```

## 6. 查询任务状态

### `GET /api/image-tasks/:task_id`

#### 响应体

```json
{
  "task_id": "task_page_003_v1",
  "task_type": "page_image_generation",
  "status": "succeeded",
  "retry_count": 1,
  "failure_reason": null,
  "image_asset": {
    "image_id": "img_page_003_v1",
    "url": "https://example.com/assets/img_page_003_v1.png",
    "width": 1024,
    "height": 1024
  },
  "cost": {
    "provider": "example-provider",
    "model": "example-image-model",
    "input_cost": 0.02,
    "output_cost": 0.08,
    "total_cost": 0.10,
    "currency": "USD"
  }

```

## 7. 重试单页图片

### `POST /api/image-tasks/:task_id/retry`

#### 请求体

```json
{
  "retry_reason": "character_inconsistent",
  "override_scene_spec": {
    "composition": "半身中景，主角更靠近画面中心",
    "background_elements": ["少量积木"]
  }
}
```

#### 响应体

```json
{
  "task_id": "task_page_003_v2",
  "previous_task_id": "task_page_003_v1",
  "status": "queued"
}
```

## 任务状态机

### 状态枚举

- `queued`
- `running`
- `succeeded`
- `failed`
- `needs_review`
- `cancelled`

### 状态说明

- `queued`
  - 已创建，等待调度
- `running`
  - 正在调用模型或处理中
- `succeeded`
  - 生成完成，可直接使用
- `failed`
  - 生成失败，且未自动恢复
- `needs_review`
  - 技术上已生成成功，但需人工确认
- `cancelled`
  - 用户或系统主动取消

## 失败原因枚举

建议统一失败原因，便于统计。

- `character_inconsistent`
- `scene_mismatch`
- `composition_mismatch`
- `quality_artifact`
- `unsafe_content`
- `provider_error`
- `timeout`
- `unknown`

## 自动重试策略

### 可自动重试的情况

- `provider_error`
- `timeout`
- 短暂网络错误

### 不建议自动重试的情况

- `character_inconsistent`
- `scene_mismatch`
- `composition_mismatch`

这些更适合带着调整参数重试，而不是原样重跑。

## 降级策略

当单页连续失败时，可执行：

1. 减少背景元素
2. 降低配角数量
3. 切换到默认稳定构图
4. 切换到默认稳定风格
5. 标记为 `needs_review`

## 前端联调建议

前端至少要支持：

- 角色图生成任务轮询
- 单页任务状态展示
- 整本任务总进度展示
- 单页失败后重试入口
- 单页替换图片入口
- 成本字段可选展示给内部运营后台

## 埋点和监控指标

建议追踪：

- 单页首次成功率
- 单页平均重试次数
- 单本读本平均总成本
- 不同风格的失败率
- 不同失败原因占比
- 角色图通过率
- 可直接交付率

## 验收标准

接口设计是否合格，至少看这些点：

- 能否独立生成角色图
- 能否独立生成单页图
- 能否批量生成整本图
- 能否只重试单页
- 能否追踪单次图片成本
- 能否统计失败原因和稳定性指标
