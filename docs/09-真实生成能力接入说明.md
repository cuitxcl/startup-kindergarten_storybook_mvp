# Kindleaf 真实生成能力接入说明

## 1. 当前结论

演示版已经冻结，下一阶段不再继续扩大 mock 功能范围，重点转向真实生成能力。

当前后端已经具备 provider 边界：

- 文本生成：DeepSeek HTTP provider。
- 插图生成：字节跳动 Seedream 图片 provider，作为项目默认真实生图模型。
- 组合模式：DeepSeek 负责文本，Seedream 负责插图。
- 默认模式：未配置真实密钥时回退到结构化 mock provider。

当前仍未生产化的是：

- 真实 DeepSeek + Seedream 组合试运行。2026-07-21 已运行 `./scripts/check-real-provider-readiness.sh --composite --allow-missing-keys`，脚本已能从 `.env` 读取本地 DeepSeek key，本地环境、endpoint、PostgreSQL、端口和 storage 检查均通过；DeepSeek provider 已支持 `DEEPSEEK_ENDPOINT_PATH` 路径或完整 URL 覆盖；真实 DeepSeek 文本 smoke 已通过故事方案、角色、分页、定制方案、写回绘本、成本账本和敏感输入脱敏审计验收；当前组合模式缺口是未配置 `SEEDREAM_API_KEY` 或 `ARK_API_KEY`，因此还没有消耗真实 Seedream 额度跑端到端图片生成。
- 真实 Seedream 单页图片试运行。2026-07-21 已验证缺 key 分支：`scripts/smoke-real-seedream-image.sh` 在缺少 `SEEDREAM_API_KEY` 或 `ARK_API_KEY` 时会停在 readiness 阶段并返回 code 2，不会启动后端或调用真实图片模型。
- 真实生成 prompt 质量。
- 完整内容安全、图片审核和人工复核。
- 真实 provider usage 精确成本统计。
- 外部服务限流和退避。
- 对象存储和 CDN。当前已抽出本地 storage service 作为替换边界，但还没有接真实 OSS/S3/CDN。
- 真实 PDF 的专业排版质量和文件生命周期。

当前已经具备的生产化基础：

- 真实 provider smoke 和 readiness 脚本会自动读取仓库根目录或 `server/` 下的 `.env.local`、`.env`，并保持命令行环境变量优先，便于本地放置 DeepSeek、Seedream 或 ARK key。`.env` 与 `.env.local` 已被 `.gitignore` 忽略。真实 DeepSeek、真实 Seedream 和真实组合 smoke 会尊重 `DEEPSEEK_ENDPOINT_PATH`、`SEEDREAM_ENDPOINT_PATH`、`ARK_IMAGE_ENDPOINT_PATH` 的覆盖配置，支持绝对路径或完整 URL；Seedream 检查也会按后端实际读取规则识别 `ARK_BASE_URL` 和 `ARK_IMAGE_MODEL`，避免使用火山方舟命名时被默认配置误判。DeepSeek 和 Seedream 的 base URL、endpoint、model、图片尺寸等配置只把非空字符串视为有效配置，空值会继续回退到别名或默认值。
- 真实文本 provider 外发前会做最小输入脱敏：儿童 ID、昵称、家长邮箱、电话、家庭信息、医疗/诊断/身份类字段会被替换为 redacted 标记。DeepSeek 文本任务输出会记录不含原文的 `privacy_audit.redacted` 和 `labels`，用于说明输入是否发生过脱敏以及命中的类别。
- 文本 provider 输出写回绘本前会做基础敏感信息扫描，命中明显邮箱、手机号、身份、住址或医疗信息时任务失败，不会把风险文本写入绘本。
- Seedream 图片 provider 外发前会对插图 prompt 做最小脱敏：UUID、邮箱、手机号、身份/住址/家庭/医疗类明显敏感词会被替换为 redacted 标记；长编号或时间戳不会被误判为手机号。任务输出会记录不含原文的 `image.privacy_audit.redacted` 和 `labels`，用于说明是否发生过脱敏以及命中的类别。
- PDF 导出、分享链接创建和公开分享导出前会做本地交付隐私扫描，阻断明显邮箱、手机号、身份、住址和医疗信息进入 PDF 或公开分享链路，并写入不含原文的交付拦截审计。
- 生成任务成功或失败后会写入 `generation_cost_logs` 估算成本账本。
- DeepSeek 响应中的 `usage` 会保留到任务输出的 `provider_usage`，成本账本会优先使用 `prompt_tokens/completion_tokens`；没有 usage 时才回退到本地粗估。
- 成本估算单价已支持环境变量配置，默认仍保持演示估算值。
- 可通过 `KINDLEAF_COST_BUDGET_LIMIT_MICROS` 配置生成预算上限，通过 `KINDLEAF_COST_BUDGET_WARNING_PERCENT` 配置预算预警线；达到预警线时运营端会提示，达到上限后，后端会拒绝新建生成任务和失败任务重试。
- 平台运营端可通过 `/api/operator/generation-costs` 查看成本汇总和明细，前端平台市场管理页已经展示“AI 生成成本账本”。
- PDF/PNG 文件读写已集中到后端 storage service，当前实现仍是本地磁盘，后续接对象存储时优先保持现有权限下载 API 不变。

## 2. Provider 配置矩阵

| 场景 | 环境变量 | 文本任务 | 插图任务 | 适用阶段 |
|---|---|---|---|---|
| 演示 mock | `KINDLEAF_GENERATION_PROVIDER=mock` | mock | mock | 本地演示 |
| 真实文本 | `KINDLEAF_GENERATION_PROVIDER=deepseek` + `DEEPSEEK_API_KEY` | DeepSeek | mock | 文本生成试接入 |
| 真实插图 | `KINDLEAF_GENERATION_PROVIDER=seedream` + `SEEDREAM_API_KEY` 或 `ARK_API_KEY` | mock | Seedream | 图片生成默认验收 |
| 文本 + 插图组合 | 不设置 provider，配置 `DEEPSEEK_API_KEY` + `SEEDREAM_API_KEY` 或 `ARK_API_KEY` | DeepSeek | Seedream | 真实生成最小闭环 |

图片生成方向已固定为 Seedream。推荐下一阶段先跑 Seedream 单页插图真实验收，再跑 DeepSeek+Seedream 组合模式；文本质量可以继续单独调 DeepSeek prompt。

## 3. 已支持的文本任务

DeepSeek provider 覆盖：

- `storybook_plan`：故事方案。
- `storybook_roles`：角色与道具。
- `storybook_pages`：分页图文。
- `customization_plan`：定制方案。

provider 输出统一写入 `generation_jobs.output_json`：

```json
{
  "schema_version": "generation.provider.v1",
  "provider": "deepseek",
  "mode": "storybook_plan",
  "message": "绘本方案已生成"
}
```

角色和分页任务成功后，后端会把结构化结果写回绘本角色和分页数据。

## 4. 已支持的插图任务

Seedream 图片 provider 覆盖：

- `storybook_page_image`：单页插图生成或重绘。

输出会写入本地 PNG 文件，并通过登录态 API 下载：

```text
/api/workspaces/{workspace_id}/generation-jobs/{job_id}/image
```

当前还不是生产文件方案。文件读写已经收敛到 storage service，但生产化前仍需要接真实对象存储、访问控制策略和图片审核。
Seedream 请求前会先脱敏插图 prompt，避免老师在插图描述里误填的联系方式、身份、住址、家庭或医疗信息直接外发给图片模型；任务输出里的 `image.prompt` 也使用脱敏后的 prompt，便于前端展示和审计。任务输出还会记录 `image.privacy_audit`，只包含是否脱敏和类别标签，例如 `phone/email/private_detail`，不写回原始敏感文本。
Seedream 输出写入前会校验 PNG 签名；如果 provider 返回非 PNG 内容，任务会进入不可重试 `failed`，不会把错误文本或 HTML 写成插图文件，并会在成本账本中按失败任务记录 0 成本。Seedream HTTP 失败或服务端错误会进入可重试 `failed`，保留 `next_run_at` 便于后续自动或人工重试，失败成本同样记录为 0。

## 5.1 生成能力就绪检查

在接真实密钥前，先看生成能力摘要，确认当前后端理解的 provider 配置是否符合预期：

```text
GET /api/workspaces/{workspace_id}/generation-provider
GET /api/operator/generation-provider
```

响应不会暴露密钥值，但会返回 `components[]`：

- `deepseek` 文本组件：模型、endpoint、是否配置 key、缺失配置、支持的文本任务。
- `seedream` 图片组件：模型、endpoint、是否配置 key、缺失配置、支持的图片任务。

API smoke 已覆盖注册个人空间、园所老师空间和平台运营端的组件级状态；UI smoke 已覆盖平台运营市场页、普通绘本创建页和定制绘本生成页能看到 Seedream 图片组件、缺失配置和 endpoint。

平台运营端还提供 `GET /api/operator/readiness` 和市场页“部署前总检查”，用于把数据库连接、核心表 migration、外部访问域名、登录 token secret 与 TTL、真实生成 provider 密钥与配置、storage、预算上限和演示数据开关聚合为试点就绪摘要。本地 mock 环境会显示 `needs_attention`；配置 `KINDLEAF_AUTH_TOKEN_SECRET`、合理的 `KINDLEAF_AUTH_TOKEN_TTL_SECONDS`、DeepSeek + Seedream/ARK key、真实 HTTPS `APP_HOST`、持久化 storage、预算上限并关闭 `KINDLEAF_DEMO_SEED` 后，这里应成为试点交接前的页面级确认入口。API 侧和命令行侧的 `scripts/check-trial-readiness.sh --strict` 都会在不调用模型的情况下检查明显占位 key、DeepSeek/Seedream endpoint 与 model，避免 key 已配置但仍是示例值，或 provider 地址/模型名写错。

交接前也可以通过智能检查入口运行同一个硬门禁：

```sh
./scripts/check-smart.sh trial-strict
```

如果只是验证门禁逻辑的正向分支，可以运行不启动服务、不调用模型的自测：

```sh
./scripts/check-smart.sh trial-positive
```

试点环境变量建议从仓库根目录的 `.env.trial.example` 复制到 `.env.local` 后填写。该模板默认使用 DeepSeek + Seedream 自动组合模式、非本地 HTTPS `APP_HOST`、签名 token secret、持久化 storage 和生成预算上限，目标是让 `trial-strict` 的 7 类常见阻断项都有明确填写位置。

## 6. 真实 provider 试运行步骤

### 6.1 只验证真实文本

推荐先使用脚本跑文本生成试运行。这个脚本会创建临时 PostgreSQL 库、启动测试后端，先检查运营端 `generation-provider` 摘要中 DeepSeek 文本组件已经 `ready`，再生成一本临时普通绘本和一个儿童档案，依次调用真实 DeepSeek 生成故事方案、角色设定、分页图文和定制方案，并校验角色/分页结果能写回绘本。脚本会调用真实 provider，可能消耗额度；没有 `DEEPSEEK_API_KEY` 时会直接退出。

```sh
DEEPSEEK_API_KEY=your-api-key ./scripts/smoke-real-deepseek-text.sh
```

成功后会输出：

```text
real_deepseek_plan_job=...
real_deepseek_roles_job=...
real_deepseek_pages_job=...
real_deepseek_text_applied=...
real_deepseek_customization_job=...
real_deepseek_costs=...
```

也可以手动启动后端后从页面验证：

```sh
cd server
DATABASE_URL=postgres://postgres:postgres@127.0.0.1:55432/kindleaf_development \
KINDLEAF_DEMO_SEED=1 \
KINDLEAF_GENERATION_PROVIDER=deepseek \
DEEPSEEK_API_KEY=your-api-key \
DEEPSEEK_ENDPOINT_PATH=/chat/completions \
DEEPSEEK_MODEL=deepseek-v4-flash \
  cargo run --features db -- start
```

验收重点：

- 创建普通绘本时，故事方案、角色、分页任务能返回 `provider=deepseek`。
- 定制绘本方案能返回 `provider=deepseek`。
- 发给 provider 的输入不应包含儿童 ID、儿童昵称、家长联系方式、家庭或医疗类原始字段。
- 定制绘本等命中敏感字段的任务输出应包含 `privacy_audit.redacted=true` 和类别标签，但不包含原始敏感文本。
- provider 失败时任务状态变为 `failed`，页面能看到失败原因。
- provider 输出如果包含明显联系方式、身份、住址或医疗信息，任务应失败且不写回绘本。
- 点击重试后复用原任务输入。
- 任务完成后，运营端 `/api/operator/generation-costs` 能看到对应 provider、任务类型和估算成本记录。

### 6.2 只验证真实插图

推荐先使用脚本跑单页插图试运行。这个脚本会创建临时 PostgreSQL 库、启动测试后端，先检查运营端 `generation-provider` 摘要中 Seedream 图片组件已经 `ready`，再生成一本临时普通绘本、调用字节跳动 Seedream 生成一页插图，通过登录态 API 下载 PNG，验证未登录和跨空间用户不能下载图片，并确认运营端成本账本中出现本次 `seedream/storybook_page_image` 成功记录。脚本会调用真实 provider，可能消耗额度；没有 `SEEDREAM_API_KEY` 或 `ARK_API_KEY` 时会直接退出。

真实 Seedream 脚本会自动先做不消耗额度的前置检查；也可以提前单独运行。检查输出会包含 provider 配置、PostgreSQL、端口、storage 目录、文件大小上限和文件名安全规则：

```sh
./scripts/check-real-provider-readiness.sh --seedream
```

```sh
SEEDREAM_API_KEY=your-api-key ./scripts/smoke-real-seedream-image.sh
```

如果团队统一使用火山方舟命名，也可以改用：

```sh
ARK_API_KEY=your-api-key ./scripts/smoke-real-seedream-image.sh
```

成功后会输出：

```text
real_seedream_image_job=...
real_seedream_storybook=...
real_seedream_image=/path/to/.tmp/real-seedream-image.png
```

也可以手动启动后端后从页面验证：

```sh
cd server
DATABASE_URL=postgres://postgres:postgres@127.0.0.1:55432/kindleaf_development \
KINDLEAF_DEMO_SEED=1 \
KINDLEAF_GENERATION_PROVIDER=seedream \
SEEDREAM_API_KEY=your-api-key \
SEEDREAM_ENDPOINT_PATH=/api/v3/images/generations \
SEEDREAM_IMAGE_MODEL=doubao-seedream-5-0-lite \
  cargo run --features db -- start
```

Seedream provider 默认使用方舟兼容路径 `/api/v3/images/generations`。如果后续切到其他火山网关，可通过 `SEEDREAM_ENDPOINT_PATH` 或 `ARK_IMAGE_ENDPOINT_PATH` 覆盖；也可以直接把完整 endpoint URL 填到 `SEEDREAM_ENDPOINT_PATH`。如果团队统一使用火山方舟命名，`ARK_BASE_URL`、`ARK_IMAGE_ENDPOINT_PATH`、`ARK_IMAGE_MODEL` 会被后端和 readiness 脚本共同识别。

配置读取遵循“非空优先”规则：`SEEDREAM_*` 有非空值时优先生效；为空时继续读取对应 `ARK_*` 别名；两者都为空时使用默认方舟地址、`/api/v3/images/generations` 和 `doubao-seedream-5-0-lite`。这让 `.env.example` 中保留空别名不会覆盖默认真实生图配置。

Seedream/ARK key 到手后的最短验收清单：

1. 把真实 key 放到仓库根目录 `.env.local` 或当前 shell 环境，二选一即可：

   ```sh
   SEEDREAM_API_KEY=真实-key
   ```

   或：

   ```sh
   ARK_API_KEY=真实-key
   ```

2. 先做不消耗额度的本地前置检查：

   ```sh
   ./scripts/check-real-provider-readiness.sh --seedream
   ```

   这一步应看到 Seedream/ARK key 已配置、endpoint 合法、model 非空、PostgreSQL 可用、测试端口空闲、storage 可写。明显占位 key 会被当作缺失 key。

3. 再跑一次真实单页图片 smoke：

   ```sh
   RUN_SEEDREAM=required RUN_DEEPSEEK=skip RUN_COMPOSITE=skip ./scripts/smoke-real-providers.sh
   ```

   这一步会真正调用 Seedream，可能消耗额度；成功后会验证 PNG 下载、未登录拒绝、跨空间拒绝和 `seedream/storybook_page_image` 成本账本。

4. 如果 DeepSeek key 也已经配置，再跑真实组合 smoke：

   ```sh
   RUN_DEEPSEEK=skip RUN_SEEDREAM=skip RUN_COMPOSITE=required ./scripts/smoke-real-providers.sh
   ```

   这一步会验证“DeepSeek 生成故事/角色/分页 + Seedream 生成第一页插图”的最小真实生成闭环。

验收重点：

- 绘本详情页创建插图任务。
- 任务返回 `provider=seedream`。
- 图片只能通过登录态 API 下载。
- 未登录用户下载图片返回 `401`，跨空间用户下载图片返回 `403`。
- 图片任务完成后，脚本会强制检查运营端成本账本中存在本次 `seedream/storybook_page_image` 成功记录，且 `image_count=1`。

### 6.3 验证文本 + 插图组合模式

推荐先使用脚本跑真实组合试运行。这个脚本会创建临时 PostgreSQL 库、启动测试后端，先检查运营端 `generation-provider` 摘要中 DeepSeek 文本组件和 Seedream 图片组件都已经 `ready`，再生成一本临时普通绘本，依次调用真实 DeepSeek 生成故事方案、角色设定和分页图文，最后调用真实 Seedream 生成第一页插图，下载 PNG，验证未登录和跨空间用户不能下载图片，并检查运营端成本账本。脚本会调用真实 provider，可能消耗额度；缺少 `DEEPSEEK_API_KEY`，或缺少 `SEEDREAM_API_KEY` / `ARK_API_KEY` 时会直接退出。

真实组合脚本会自动先确认本机 Docker/PostgreSQL、端口、storage 目录、文件大小上限、文件名安全规则和 provider 配置格式都满足要求；也可以提前单独运行：

```sh
./scripts/check-real-provider-readiness.sh --composite
```

```sh
DEEPSEEK_API_KEY=your-deepseek-key SEEDREAM_API_KEY=your-seedream-key ./scripts/smoke-real-composite-provider.sh
```

如果团队统一使用火山方舟命名，也可以改用：

```sh
DEEPSEEK_API_KEY=your-deepseek-key ARK_API_KEY=your-ark-key ./scripts/smoke-real-composite-provider.sh
```

成功后会输出：

```text
real_composite_provider=deepseek+seedream/...
real_composite_plan_job=...
real_composite_roles_job=...
real_composite_pages_job=...
real_composite_text_applied=...
real_composite_image_job=...
real_composite_image=/path/to/.tmp/real-composite-seedream-image.png
real_composite_costs=...
```

也可以手动启动后端后从页面验证：

```sh
cd server
DATABASE_URL=postgres://postgres:postgres@127.0.0.1:55432/kindleaf_development \
KINDLEAF_DEMO_SEED=1 \
DEEPSEEK_API_KEY=your-api-key \
SEEDREAM_API_KEY=your-api-key \
  cargo run --features db -- start
```

不显式设置 `KINDLEAF_GENERATION_PROVIDER` 时，后端会根据密钥自动组合：

- 文本：DeepSeek。
- 插图：Seedream。

组合模式下，任务失败记录会按任务类型标记具体 provider：文本任务失败标记为 `deepseek`，插图任务失败标记为 `seedream`，整体 provider 摘要显示 `deepseek+seedream`。

组合模式下，成本记录同样按具体任务 provider 归因：文本任务归 `deepseek`，图片任务归 `seedream`。

验收重点：

- 运营端 `generation-provider` 摘要返回 `provider=deepseek+seedream`、`mode=composite`。
- DeepSeek 文本组件和 Seedream 图片组件都 `configured=true`、`ready=true`。
- 普通绘本的方案、角色和分页由 DeepSeek 生成，并写回真实绘本。
- 第一页插图由 Seedream 生成，返回登录态 PNG 下载地址。
- 未登录用户下载插图返回 `401`，跨空间用户下载插图返回 `403`。
- 运营端成本账本同时出现 DeepSeek 文本任务和 Seedream 图片任务。
- 成本账本里同时出现 DeepSeek 和 Seedream 成功任务。

## 7. 本地文件存储配置

PDF 导出和 Seedream/mock 插图当前仍使用本地文件系统，但读写已经集中到 storage service。默认目录为：

```sh
tmp/exports
tmp/generated-images
```

试点或部署环境可以把它们改到持久化挂载目录，现有 API 下载路径不变：

```sh
KINDLEAF_STORAGE_ROOT=/var/lib/kindleaf
# 可选：分别覆盖 PDF 和插图目录
KINDLEAF_EXPORTS_DIR=/var/lib/kindleaf/exports
KINDLEAF_GENERATED_IMAGES_DIR=/var/lib/kindleaf/generated-images
KINDLEAF_EXPORT_MAX_BYTES=52428800
KINDLEAF_GENERATED_IMAGE_MAX_BYTES=15728640
```

部署机器上需要先创建目录，并把权限交给后端运行用户：

```sh
sudo mkdir -p /var/lib/kindleaf/{exports,generated-images}
sudo chown -R <kindleaf-runtime-user>:<kindleaf-runtime-group> /var/lib/kindleaf
```

默认 PDF 上限为 50MB，生成插图 PNG 上限为 15MB；对应变量设为 `0` 时关闭该上限。这一步仍不是对象存储生产化，只是先把本地演示目录、文件名安全和文件大小边界从业务代码中解耦。下一阶段接对象存储时，应继续保持登录态下载 API 和公开分享下载 API 不变，只替换 storage service 的底层实现。

## 8. 成本估算配置

当前成本账本仍是估算口径，不等同于账单金额。可通过环境变量调整估算单价：

```sh
KINDLEAF_COST_CURRENCY=USD
KINDLEAF_COST_DEEPSEEK_INPUT_UNIT_MICROS=1
KINDLEAF_COST_DEEPSEEK_OUTPUT_UNIT_MICROS=4
KINDLEAF_COST_SEEDREAM_IMAGE_MICROS=40000
KINDLEAF_COST_BUDGET_LIMIT_MICROS=2000000
KINDLEAF_COST_BUDGET_WARNING_PERCENT=80
```

说明：

- `*_MICROS` 表示一百万分之一货币单位。
- DeepSeek 文本任务优先使用 `provider_usage.prompt_tokens / completion_tokens` 计算。
- 没有 provider usage 时，文本任务回退到本地 JSON 长度粗估。
- Seedream 图片任务当前按成功图片数量估算。
- `KINDLEAF_COST_BUDGET_LIMIT_MICROS` 为空时不启用预算拦截；设置为正整数后，当前空间成功任务估算成本达到或超过预算时，新建生成任务和失败任务重试会返回 `409 state_conflict`。
- `KINDLEAF_COST_BUDGET_WARNING_PERCENT` 默认 80，支持 1-100 之间的数值；当前空间预算使用率达到该阈值后，成本汇总会返回 `budget_warning=true`，前端运营页展示“生成预算接近上限”。

## 9. 下一步最小真实闭环

建议按以下顺序推进：

1. **真实 Seedream 单页插图验收**
   先运行 `scripts/check-real-provider-readiness.sh --seedream`，再用 `scripts/smoke-image-provider.sh` 和 `scripts/smoke-real-seedream-image.sh` 各跑一次，确认外发 prompt 已脱敏、返回 PNG 合法、图片能通过登录态 API 下载。

2. **真实文本写回验收**
   确认角色和分页结果写回绘本，并且页面能继续编辑。

3. **DeepSeek+Seedream 组合验收**
   同时配置 `DEEPSEEK_API_KEY` 和 `SEEDREAM_API_KEY` 或 `ARK_API_KEY`，先运行 `scripts/check-real-provider-readiness.sh --composite`，再运行 `scripts/smoke-real-composite-provider.sh`，跑通普通绘本的文本生成、角色分页写回、单页 Seedream 插图生成、PNG 下载和成本账本。

4. **失败恢复验收**
   先用 `scripts/smoke-generation-provider-failure.sh` 验证 fake provider 500、非法 JSON 内容、缺少任务必需结构、成本记录和重试成功；再用错误 key 对真实 provider 做一次人工验证，确认页面错误摘要和重试入口表现正确。

5. **成本账本验收**
   确认成功、失败、文本、图片任务都会写入成本日志，DeepSeek 文本任务会保留 `provider_usage`，运营端可以按空间、provider、任务类型和状态筛选；运行 `scripts/smoke-generation-budget.sh` 验证预算预警、超限摘要和生成拦截。

6. **真实插图试运行**
   只跑单页插图，控制成本，不进入批量生成。

7. **文件存储改造**
   当前已经抽出本地 storage service；下一步把 storage 实现替换为对象存储，保留现有下载 API。

8. **PDF 排版升级**
   现有本地基础 PDF 已有封面、逐页正文、插图画框、插图提示、页脚，并会嵌入每页最新成功生成的 PNG 插图；下一步继续补专业排版模板、字体嵌入、封面设计和对象存储。

## 10. 生产化前必须补齐

- Prompt 版本管理。
- 输入脱敏审计，尤其是儿童资料和老师备注。
- 文本输出已有基础敏感信息拦截；仍需补模型内容安全审核、图片审核和人工复核。
- 更多 provider 的 usage 映射、真实额度同步、预算通知告警和账单关联。
- 单空间并发限制。
- provider 超时、限流、退避和死信队列。
- 对象存储访问权限。
- 真实 PDF 文件生命周期和对象存储。
- 运营后台按空间、任务类型、状态查看生成任务和真实成本。

## 11. 不建议继续做的事

- 不再增加新的 mock 业务页面。
- 不再把“页面能点”当作真实生成完成。
- 不再用单次手动成功替代 provider smoke。
- 不在前端保存儿童完整敏感资料作为生成任务 payload。
- 不把本地 `/tmp` 文件路径作为生产下载方案。
