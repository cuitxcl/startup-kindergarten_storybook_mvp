# 普通绘本生成流程

## 定义

普通绘本是不绑定具体孩子、可供全班共读或老师教学使用的完整图文绘本。

第一版先只解决“老师生成一本普通绘本”的闭环，不引入儿童档案、家长信息、批量定制、支付和复杂共享。

## 核心原则

- 先确认绘本方案，再确认角色设定，最后生成完整绘本。
- 绘本方案先解决“这个故事讲什么、值不值得继续生成”。
- 角色设定服务于已确认的故事，不早于故事方向确认。
- 老师每一步只做低成本确认，避免生成到后面才大返工。
- 后续如果要做儿童定制，普通绘本可以作为可复用母本。

## 推荐流程

```text
1. 填写绘本需求
2. 生成绘本方案
3. 老师确认/调整方案
4. 生成角色与关键道具设定
5. 老师确认/修改角色
6. 生成分页图文内容
7. 老师编辑分页内容
8. 生成插图
9. 预览、保存、导出
```

## 步骤说明

### 1. 填写绘本需求

老师输入最少必要信息：

| 字段 | 说明 | 建议 |
| --- | --- | --- |
| 绘本主题 | 想讲的教学或成长主题 | 必填，例如“学会分享”“午睡习惯”“入园适应” |
| 年龄段 | 目标阅读对象 | 必填，可选小班/中班/大班或 3-4/4-5/5-6 岁 |
| 页数 | 绘本长度 | 必填，默认 6 页 |
| 使用场景 | 绘本用在哪里 | 必填，课堂共读/规则引导/家园沟通/活动导入 |
| 故事风格 | 故事表达方式 | 可选，温暖/幽默/生活化/童话 |
| 画面风格 | 插图风格 | 第一版可默认，不必复杂配置 |

### 2. 生成绘本方案

系统先生成轻量方案，而不是直接生成完整绘本。

绘本方案应包含：

| 内容 | 作用 |
| --- | --- |
| 绘本标题 | 让老师快速判断方向 |
| 一句话故事简介 | 判断故事是否符合主题 |
| 教学目标 | 判断是否适合课堂使用 |
| 情绪/行为目标 | 明确孩子要理解或练习什么 |
| 页数结构 | 每页大概讲什么 |
| 角色需求 | 需要主角、朋友、老师、家长还是想象角色 |
| 关键道具/场景 | 例如玩具、沙漏、教室、操场 |
| 画面基调 | 例如温暖、明亮、生活化 |

注意：这一阶段只生成“角色需求”，不生成详细外观。

示例：

```text
标题：《一起玩小汽车》

故事简介：
一个小动物主角很喜欢自己的小汽车，起初不愿意和朋友轮流玩。
在老师引导下，它尝试表达感受、轮流等待，最后发现一起玩也很快乐。

页数结构：
1. 主角带着喜欢的小汽车来到幼儿园。
2. 朋友也想玩，主角有点舍不得。
3. 老师帮助两个孩子说出自己的想法。
4. 大家尝试用沙漏轮流。
5. 主角发现等待后也能玩得开心。
6. 大家一起整理玩具，约定下次继续轮流。

角色需求：
- 1 个主角：正在学习分享的幼儿化动物角色
- 1 个配角：想一起玩的朋友
- 1 个引导者：温柔的老师角色
- 1 个关键道具：可轮流玩的玩具
```

### 3. 老师确认/调整方案

老师可以修改：

- 标题
- 故事简介
- 页数结构
- 每页摘要
- 教学目标
- 角色需求
- 关键场景或道具

老师确认方案后，系统再进入角色设定生成。

如果方案不对，老师应该在这里重新生成或调整，而不是等完整正文和插图生成后再返工。

### 4. 生成角色与关键道具设定

系统根据已确认的绘本方案生成具体角色。

角色设定包含：

| 字段 | 示例 |
| --- | --- |
| 角色名 | 小兔米米 |
| 角色类型 | 动物主角 |
| 外观描述 | 白色小兔，圆脸，长耳朵 |
| 服装/代表颜色 | 黄色背带裤 |
| 性格 | 有点害羞，但愿意尝试 |
| 故事作用 | 学习分享玩具的主角 |
| 是否重复出现 | 是 |
| 是否需要插图一致性 | 是 |

关键道具设定包含：

| 字段 | 示例 |
| --- | --- |
| 道具名 | 红色小汽车 |
| 外观描述 | 圆润安全的红色玩具车 |
| 故事作用 | 引发轮流和分享的核心道具 |
| 是否重复出现 | 是 |

### 5. 老师确认/修改角色

老师可以：

- 修改角色名
- 修改角色类型
- 修改外观描述
- 修改服装和代表颜色
- 修改性格
- 替换某个角色
- 添加或删除非必要配角
- 修改关键道具

角色确认后，后续正文和插图都使用这套角色设定。

第一版建议规则：

- 角色确认后仍允许修改。
- 修改角色后，系统标记相关插图需要重新生成。
- 不自动重写所有正文，除非老师明确选择重新生成。

### 6. 生成分页图文内容

系统基于“已确认方案 + 已确认角色”生成完整分页内容。

每页包含：

- 页面标题
- 页面正文
- 插图描述
- 出现角色
- 场景和动作
- 教师提问
- 教师提示

### 7. 老师编辑分页内容

老师可以逐页编辑：

- 页面标题
- 正文
- 插图描述
- 教师提问
- 教师提示

这一阶段主要做细节修正，不再大幅改变故事结构。

如果老师要改变故事结构，应返回“绘本方案”重新调整。

### 8. 生成插图

插图生成使用：

- 全局画风
- 当前页场景
- 当前页动作
- 当前页出现角色
- 已确认角色设定
- 已确认关键道具设定

第一版至少支持：

- 单页生成
- 单页重绘
- 查看生成状态
- 失败重试

如果暂时不接真实图片生成，也可以先生成插图描述和 prompt，保留后续接入空间。

### 9. 预览、保存、导出

老师最终可以：

- 预览整本绘本
- 保存为普通绘本
- 导出 PDF
- 复制生成新版本
- 后续作为普通绘本母本派生儿童定制版本

## 状态设计

| 状态 | 含义 |
| --- | --- |
| 需求填写中 | 老师还没有生成绘本方案 |
| 方案待确认 | 系统已生成方案，老师未确认 |
| 方案已确认 | 可生成角色设定 |
| 角色待确认 | 系统已生成角色，老师未确认 |
| 角色已确认 | 可生成分页图文内容 |
| 正文编辑中 | 已生成分页正文，老师可逐页编辑 |
| 插图生成中 | 正在生成或重绘插图 |
| 可导出 | 内容已可预览并导出 |

## 第一版范围

必须做：

- 新建普通绘本入口
- 需求填写
- 生成绘本方案
- 老师确认/修改方案
- 生成角色与关键道具设定
- 老师确认/修改角色
- 生成分页正文
- 逐页编辑
- 保存普通绘本
- 导出 PDF

可以延后：

- 真实图片生成
- 角色参考图
- 复杂画风选择
- 多版本对比
- 班级批量定制
- 儿童档案绑定
- 家长分享
- 支付

## 关键判断

普通绘本生成流程应定为：

```text
先确认绘本方案，再确认角色设定，最后生成完整绘本。
```

这样能最大限度减少返工，也能为后续“从普通绘本派生儿童定制版”留下清晰结构。
