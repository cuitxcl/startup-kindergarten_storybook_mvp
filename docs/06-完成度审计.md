# Kindleaf 完成度审计

## 1. 审计目的

把当前项目从“感觉差不多”变成“证据上说得清”。

这份文档只审计一个问题：

> Kindleaf 当前是否已经完成了“老师可登录、可进空间、可创建普通绘本、可维护儿童档案、可派生定制绘本、可导出/分享/投稿/复用”的可演示闭环。

## 2. 当前证据

以下内容已经有代码、脚本或测试证据。结论按“本地可演示闭环”口径判断，不等同于生产上线。

### 2.1 自动化与验证入口

- 前端构建命令：`npm --prefix frontend run build`
- 后端测试命令：`cargo test --manifest-path server/Cargo.toml --quiet`
- 数据库特性检查：`cargo check --manifest-path server/Cargo.toml --features db`
- 本地快速检查入口：`scripts/check-fast.sh`
- API smoke 脚本：`scripts/smoke-api.sh`
- 临时库 API smoke 脚本：`scripts/smoke-api-temp-db.sh`
- UI smoke 脚本：`scripts/smoke-ui.mjs`
- 轻量脚本校验：`bash -n scripts/smoke-api.sh`、`node --check scripts/smoke-ui.mjs`
- 标准演示数据入口：`cargo run --features db -- db seed`
- migration 回滚/重放验收：`scripts/check-migrations.sh`
- 生成预算拦截专项验收：`scripts/check-smart.sh budget`
- 空库完整验收：临时 PostgreSQL 库执行 `migration up`、连续两次 `db seed`、再运行 `scripts/smoke-all.sh` 已通过

说明：

- 完整 smoke 需要 PostgreSQL 和浏览器运行环境；`scripts/smoke-all.sh` 会先执行 migration，再自动启动临时后端和 API 模式前端。
- `scripts/check-fast.sh` 不启动服务、不创建数据库，适合作为每次小步改动后的快速 gate；它不能替代完整 API/UI smoke。

- 公开导出同样可通过 `file_url` 下载 PDF 文件
- 分享链接撤销后，分享详情、公开导出创建、公开导出查询都会返回不可访问
- API 模式下绘本详情读取失败时不会跳转到 mock 绘本或同标题别名
- API 模式下普通绘本向导完成页不会把失败创建的绘本导向 mock 详情

### 2.5 定制绘本闭环

已验证内容：

- 可基于普通绘本生成定制副本
- 定制流程可在前后端演示链路中走通
- 定制方案会作为 generation output 在前端展示
- 定制副本请求可以携带 customization plan
- 后端会把定制策略写入派生绘本的教学目标上下文
- API 模式下定制源绘本读取失败时不会自动改用 mock 母本
- API 模式下定制副本生成结果不会在没有真实副本 ID 时跳回源绘本

### 2.6 园所、投稿与市场

已验证内容：

- 可管理园所成员
- 可管理班级
- 班级创建会拒绝空字段和重复班级名，避免老师授权、儿童分班和后续班级统计出现歧义
- 空班级可由园所管理员归档；仍有儿童档案的班级不可归档，归档后班级不再出现在当前使用班级列表和授权选项
- 可创建投稿草稿
- 可做隐私确认
- 可进入市场审核与复制流程
- 同一空间的同一来源绘本不能重复投稿；API smoke 覆盖重复创建投稿返回 `409 state_conflict`
- 审核通过后市场模板可带 source storybook 关系
- 运营审核后前端可跳转查看市场管理
- API 模式下市场模板详情读取失败时不会自动改用 mock 模板

## 4. 结论

从“本地可演示闭环”这个目标看，项目核心主线已经成立，并已冻结为本地演示版；但它还没有达到“真实试点可用”或“生产上线”的完成度。

当前更准确的判断是：

- 本地可演示闭环：约 94%-95%，已通过完整 smoke，可冻结演示
- 产品方向和前端原型：约 85%-90%
- 后端骨架和 API：约 85%-90%
- 数据库与真实持久化闭环：约 80%-85%
- 前后端真实联调：约 85%-90%
- 真实 AI 生成能力：约 73%-74%，DeepSeek 文本已真实验收，Seedream/ARK 图片 key 仍缺失；试点 readiness 已具备脚本和运营页双入口
- 生产化能力：约 24%-29%，已具备最小容器运行材料、容器 healthcheck、数据库备份恢复脚本和 readiness 门禁，但仍缺正式部署、HTTPS、对象存储、外部监控告警、定时异地备份和发布流程

预计节奏：

- 若目标是完整本地演示：当前已可冻结，后续只做演示前小修。
- 若目标是小范围园所试用：还需要约 3-5 周。
- 若目标是接近生产上线：还需要约 2-3 个月。

下一步不建议继续扩大 mock 功能范围，应优先进入真实生成能力和试点生产化准备。
