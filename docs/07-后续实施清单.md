# Kindleaf 后续实施清单

## 1. 目标

当前项目已经完成并冻结“本地可演示闭环”的主线，接下来要把它往真实可用、可运营、可试点的方向继续推进。

这份清单的作用是把后续工作按优先级排出来，避免阶段目标混在一起。

## 2. 优先级顺序

### P0：演示版冻结维护

目标：保持本地演示版稳定，不再继续扩大 mock 功能范围。

## 3. 建议执行节奏

1. 先做 P0，让当前前端原型真正吃后端数据库。
2. 再做 P1，让绘本内容、插图和导出变成真实生成。
3. 然后做 P2，把系统变成可以试点的服务。
4. 最后做 P3，补商业化和规模化运营能力。
