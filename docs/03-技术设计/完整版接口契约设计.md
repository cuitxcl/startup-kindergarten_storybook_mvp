# 幼儿园个性化内容平台完整版接口契约设计

## API summary

首发版 API 需要覆盖老师端、家长提交端、图片生成端、共享端和园所管理端。接口设计必须支持异步任务、共享治理、额度管理和多角色访问。

## Resources

- `children`
  - 老师维护的孩子档案
- `parents`
  - 家长提交与身份信息
- `story_templates`
  - 系统内部使用的底层模板骨架
- `case_library`
  - 老师可直接选择的成品案例
- `character_profiles`
  - 儿童角色卡和参考图
- `storybooks`
  - 已生成的个性化读本
- `storybook_pages`
  - 读本逐页内容
- `image_tasks`
  - 图片生成任务
- `exports`
  - 读本导出动作和结果
- `shared_library`
  - 共享内容库
- `schools`
  - 园所与额度

## Endpoints

### `GET /api/children`

用途：
- 获取当前老师名下的儿童列表

支持：
- 搜索 `keyword`
- 排序默认按 `updated_at desc`

### `POST /api/children`

用途：
- 创建一个最小儿童档案

### `POST /api/parent-intakes`

用途：
- 家长提交儿童基础资料和照片

### `GET /api/parent-intakes`

用途：
- 教师查看家长提交状态

### `GET /api/cases`

用途：
- 获取可选成品案例列表

支持：
- `theme`
- `age_group`
- `content_type`

### `GET /api/cases/:id`

用途：
- 获取成品案例详情、页面效果和适用说明

### `GET /api/story-templates/:id`

用途：
- 获取底层模板骨架详情，仅系统后台或运营端使用

### `POST /api/characters`

用途：
- 创建角色卡

### `GET /api/characters/:id`

用途：
- 获取角色卡详情

### `POST /api/images/reference`

用途：
- 生成标准角色参考图

### `POST /api/storybooks/generate`

用途：
- 基于孩子和成品案例生成一本新读本

### `GET /api/storybooks`

用途：
- 获取当前老师最近读本列表

### `GET /api/storybooks/:id`

用途：
- 获取一本读本详情及全部页面

### `POST /api/storybooks/:id/replace-roles`

用途：
- 基于已生成成品内容替换主角或多个配角
- 优先复用现有内容，不默认重新生成整本
- 允许从普通绘本母本直接派生出儿童定制绘本

### `PATCH /api/storybooks/:id`

用途：
- 更新读本级字段，例如标题

### `PATCH /api/storybooks/:id/pages/:pageNumber`

用途：
- 更新单页文案或锁定状态

### `POST /api/images/pages`

用途：
- 发起单页图片生成或重绘

### `POST /api/images/storybooks`

用途：
- 发起整本图片生成

### `GET /api/image-tasks/:taskId`

用途：
- 查询图片任务状态

### `POST /api/image-tasks/:taskId/retry`

用途：
- 重试单页图片任务

### `POST /api/storybooks/:id/export/pdf`

用途：
- 触发 PDF 导出并返回下载结果

## Validation rules

- 创建儿童时：
  - `name` 必填
  - `age_group` 建议必填
  - `interest_tags` 最多建议 10 个
- 家长提交时：
  - `parent_name` 必填
  - 至少一张照片建议提交
- 获取案例时：
  - `theme` 和 `age_group` 只能使用受支持枚举
- 生成读本时：
  - `child_id` 必须存在且归当前老师所有
  - `case_id` 必须存在且状态为 `active`
  - `content_type` 必须使用平台约定枚举
  - `teaching_goal` 长度建议限制在 200 字以内
  - 生成结果必须写入主角和配角角色信息
- 角色卡创建时：
  - `child_id` 必须存在
  - 至少需要图片或结构化 traits 其一
- 角色替换时：
  - `role_key` 必须存在于当前读本的角色清单中
  - 优先使用 `reuse_first` 模式
  - 只有受影响页面才允许进入重绘或重生成
  - 如果来源是普通绘本母本，优先派生出新的定制绘本，而不是覆盖原母本
- 更新页面时：
  - `pageNumber` 必须在模板页数范围内
  - `body_text` 不能为空字符串
  - `page_title` 长度建议不超过 60 字
- 共享设置时：
  - `platform_public` 必须经过审核
  - 平台公开前必须传入脱敏策略
- 导出 PDF 时：
  - 目标读本必须存在
  - 目标读本必须归当前老师所有

## Auth and permissions

- 教师接口默认要求教师身份登录
- 园所管理接口要求管理员角色
- 家长提交接口允许邀请态访问
- 老师只能访问自己授权范围内的 `children` 和 `storybooks`
- 成品案例是老师端可读资源，老师不可直接覆盖原案例
- 底层模板骨架是平台级内部资源，老师端不直接暴露
- 共享库内容按共享范围控制可见性

## Error model

统一错误包络建议：

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "请求参数校验失败",
    "details": [
      {
        "field": "name",
        "message": "姓名不能为空"
      }
    ]
  }
}
```

建议错误码：

- `UNAUTHORIZED`
  - 未登录或登录失效
- `FORBIDDEN`
  - 无访问权限
- `NOT_FOUND`
  - 资源不存在
- `VALIDATION_ERROR`
  - 字段不合法
- `CONFLICT`
  - 状态冲突，例如案例不可用或模板骨架已归档
- `TASK_FAILED`
  - 图片任务失败
- `REVIEW_REQUIRED`
  - 公开共享需审核
- `EXPORT_FAILED`
  - 导出失败
- `INTERNAL_ERROR`
  - 未预期服务端错误

## Open questions or versioning risks

- 读本生成和图片生成都已经按异步模式设计，前端必须处理任务型状态
- 如果后续加入支付链路，还需要新增订单和增值权益接口
- 如果未来共享库支持更复杂审核流，需要拆出单独审核资源
