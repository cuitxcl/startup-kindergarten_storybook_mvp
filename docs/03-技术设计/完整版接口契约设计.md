# 幼儿园个性化内容平台完整版接口契约设计

本文是当前正式 API 契约设计，和 `完整版数据库结构设计.md` 对齐。目标是让前端、后端、图片生成 worker 和运营后台共享同一套资源命名、状态规则、验证规则和错误模型。

接口覆盖：

- 老师工作台、内容中心、儿童档案、成品案例中心、读本编辑器、图片生成、导出分享。
- 家长资料提交与照片上传。
- 园所、班级、老师、额度和成本统计。
- 角色一致性、家长角色、关键道具、参考图、页面视觉主体。
- 图片生成任务、候选输出、审核事件、重试和成本记录。

## API Summary

### 设计原则

- 故事生成和图片生成分离：`storybooks` / `storybook_pages` 先落库，图片生成任务后置创建。
- 读本详情接口面向编辑器聚合返回，不要求前端拼接过多底层表。
- 写接口必须清楚映射到数据表、状态机和事务边界。
- 所有异步动作必须返回任务或作业状态，不能伪装成同步完成。
- 所有可重复点击的生成、导出、分享动作都支持 `Idempotency-Key`。
- 共享默认私有，扩大共享范围必须显式确认脱敏和审核规则。

### 通用约定

- Base path: `/api`
- Auth: 老师端使用登录态 token；家长提交端使用邀请 token；平台/园所后台使用具备角色的登录态 token。
- Content-Type: `application/json`，文件上传使用 `multipart/form-data` 或预签名上传流程。
- 时间格式：ISO 8601 UTC 字符串。
- ID 格式：UUID。
- 分页参数：`page` 从 1 开始，`page_size` 默认 20，最大 100。
- 列表响应统一返回 `items`、`page`、`page_size`、`total`。
- 删除默认软删除或归档，除非资源是未使用的草稿。

### 通用枚举

- `content_type`: `plain_storybook` / `custom_storybook`
- `story_status`: `draft` / `story_generating` / `story_ready` / `story_failed`
- `illustration_status`: `not_started` / `queued` / `running` / `ready` / `partial_failed` / `failed`
- `storybook_status`: `draft` / `generating` / `ready` / `exporting` / `archived`
- `share_scope`: `private` / `family` / `school` / `platform_review` / `platform_public`
- `task_status`: `queued` / `running` / `succeeded` / `failed` / `needs_review` / `cancelled`
- `failure_reason`: `character_inconsistent` / `scene_mismatch` / `composition_mismatch` / `quality_artifact` / `unsafe_content` / `provider_error` / `timeout` / `unknown`
- `review_status`: `pending` / `approved` / `rejected`

## Resources

### 组织与用户域

- `schools`: 园所边界、共享和成本归属。
- `classrooms`: 班级筛选、儿童分组和老师默认上下文。
- `teachers`: 老师、园所管理员、平台运营身份。
- `parents`: 家长身份和家长角色资料来源。
- `children`: 个性化内容的核心档案。
- `child_photos`: 儿童照片、授权状态和主照片。

### 内容域

- `story_templates`: 后台维护的结构骨架，老师端默认不直接编辑。
- `case_storybooks`: 老师可选的成品案例库。
- `storybooks`: 普通绘本、定制绘本和派生读本。
- `storybook_pages`: 逐页正文、互动提问、场景规格、当前图片。
- `storybook_exports`: PDF、翻页版、打印版导出任务。
- `storybook_share_links`: 分享链接、二维码和脱敏设置。

### 角色与一致性域

- `character_profiles`: 儿童角色卡，版本化。
- `parent_character_profiles`: 家长角色卡，版本化。
- `prop_profiles`: 多页出现或剧情关键道具卡。
- `reference_images`: 儿童、家长角色、道具的参考图。
- `storybook_roles`: 读本内角色和替换来源。
- `storybook_page_visual_subjects`: 页面上出现的角色和关键道具。

### 图片生成域

- `image_assets`: 图片、PDF、二维码等资产。
- `image_generation_tasks`: 图片生成任务和输入快照。
- `image_generation_outputs`: 任务输出候选图。
- `image_review_events`: 图片审核、选择、重试事件。
- `generation_cost_logs`: 生成成本流水。

## Validation Rules

| Endpoint | 必要验证 | 关键业务规则 |
|---|---|---|
| `GET /api/dashboard/teacher` | 登录态有效；`classroom_id` 属于当前园所 | 只返回当前老师可见的待办、最近读本和班级概览 |
| `GET /api/content-items` | 分页、排序、筛选枚举合法；孩子和班级过滤在授权范围内 | 内容中心只展示当前老师私有、同园共享和平台公开可见内容 |

### 通用验证

- 所有 UUID 必须格式正确且资源存在。
- 所有写接口必须校验当前用户是否能访问目标 `school_id`、`teacher_id`、`child_id`、`storybook_id`。
- 所有枚举只允许本文和数据库设计中声明的值。
- 字符串字段默认 trim；trim 后为空视为缺失。
- 数组类标签去重后保存；单个标签最长 20 字，默认最多 20 个。
- JSON 字段必须是对象或数组，不能提交字符串化 JSON。
- 带 `Idempotency-Key` 的接口若参数一致应返回同一结果；参数不一致返回 `IDEMPOTENCY_CONFLICT`。

### 组织与班级

- `schools.code` 若存在必须唯一，且只能由园所管理员或平台运营修改。
- `classrooms.school_id` 必须等于当前老师所在园所。
- `classrooms.teacher_id` 若存在，老师必须属于同一园所且状态为 `active`。
- 班级归档后不能作为新儿童默认班级，但历史儿童可继续引用。

### 儿童档案

- `children.name` 必填，最长 40 字。
- `age` 必须在 0 到 8 之间；`age_group` 必须是支持的年龄段。
- `primary_teacher_id` 由登录态写入，不允许前端伪造。
- `primary_parent_id` 若存在，必须已通过家长提交或老师创建并绑定到该儿童。
- `profile_completion_status = usable` 要求至少满足一项：`hair`、`usual_outfit`、`favorite_color`、非空 `interest_tags`、或已授权主照片。
- `profile_completion_status = complete` 要求儿童基础字段、至少一项视觉特征、至少一个兴趣标签、且照片授权不是 `revoked`。
- 老师只能补充教学相关信息；家长提交覆盖基础信息时需要老师 accept。

### 照片与资产

- 儿童照片必须绑定 `asset_type = child_photo` 的 `image_assets`。
- 设为主照片时，同一儿童其他照片自动 `is_primary = false`。
- `consent_status = revoked` 的照片不能用于生成新参考图。
- 上传文件类型仅允许平台支持的图片 MIME；图片大小和分辨率限制由上传服务返回。

### 案例与模板

- 老师端只能读取 `case_storybooks.status = published` 的案例。
- 创建读本时，`case_storybook_id` 必须 published；后台模板必须 active。
- 案例页数应在 6 到 10 页；后台创建案例时必须有教学目标和主题。
- 老师从案例复制时不能修改原案例，只能创建新 `storybooks`。

### 故事生成与读本

- `content_type = custom_storybook` 必须提供 `child_id`，且儿童归当前老师或当前园所授权范围。
- `content_type = plain_storybook` 可以没有 `child_id`，但必须有 `theme` 或 `teaching_goal`。
- `case_storybook_id` 和 `source_storybook_id` 至少提供一个作为生成来源；完全从零生成需要后台单独开关。
- 故事生成先写 `storybooks.story_status = story_generating`，再写页面；图片任务不得成为故事落库前置条件。
- `story_status = story_ready` 才能创建页面图片任务或整本图片任务。
- `status = ready` 要求故事 ready，且插图 ready 或产品明确允许文字版交付。
- `derivation_type` 必须和来源一致：从案例为 `from_case`，从普通绘本派生为 `from_plain_storybook`。

### 页面编辑

- `(storybook_id, page_number)` 唯一；新增页面后需要重排页码。
- `body_text` 必填，最长建议 800 字；封面页可以较短但不能为空。
- `page_title` 最长 60 字；`prompt_text` 最长 200 字；`teacher_tip` 最长 300 字。
- `scene_spec_status = ready` 时，`scene_spec_json` 必须包含 `location`、`action`、`composition` 中至少两项。
- `is_locked = true` 的页面不会被自动重写或自动重绘；老师显式 PATCH 页面仍可修改。
- 删除页面不能删除最后一页；删除后相关视觉主体软删除或重建。

### 角色与道具

- `character_profiles` 创建必须绑定儿童，且 `hair`、`body_proportion`、`age_group` 必填。
- 儿童角色卡新版本不能覆盖旧版本；显著变更必须新建版本。
- 生成角色参考图前，`visual_must_keep` 至少 3 项，或有授权主照片辅助。
- `parent_character_profiles` 必须绑定家长，可选绑定儿童；同一 `(parent_id, child_id, version)` 唯一。
- 多页出现、承担剧情推进或会被家长关注一致性的关键道具必须进入 `prop_profiles`。
- 一次性背景元素不建 `prop_profiles`，只放在 `scene_spec_json.background_elements`。
- `storybook_roles(storybook_id, role_key)` 唯一；角色替换必须更新 `storybook_roles`，并同步 `role_manifest_json` 缓存。
- 页面视觉主体必须刚好引用一个 `storybook_role_id` 或 `prop_profile_id`。

### 参考图

- `reference_images.subject_type` 必须和目标 FK 对应，且三个 subject FK 只能有一个非空。
- 同一 subject、同一 `style_id` 同时只能有一张 active 参考图。
- 设为 active 前必须 `review_status = approved`。
- 被 rejected 的参考图不能进入新图片任务的 `reference_image_ids`。

### 图片生成任务

- 创建单页图片任务要求页面存在、故事 ready、`scene_spec_status = ready`。
- 整本图片任务只为满足条件的页面创建子任务；`skip_locked_pages = true` 时跳过锁定页。
- 任务创建时必须固化 `input_snapshot_json`，至少包含：页面场景、style spec、相关角色卡、参考图、关键道具。
- `page_image_generation` 必须有 `storybook_page_id`；`storybook_image_generation` 是父任务，不直接产出图片。
- 原样自动重试只允许 `provider_error`、`timeout`；角色不一致、构图不符等需要带修改参数重试。
- `retry_count` 不能超过 `max_retries`；超过后返回 `RETRY_LIMIT_EXCEEDED`。
- 选择候选图时，必须保证 output 属于该页面最近有效任务或老师确认覆盖历史任务。
- 选择候选图后更新 `storybook_pages.current_image_asset_id` 和 `current_image_task_id`。

### 审核与成本

- `image_review_events` append-only，不允许修改历史审核事件。
- 审核拒绝必须提供 `reason_code`；人工重试建议提供 notes。
- 成本记录 append-only；不能因为任务失败或读本删除而删除。
- 成本查询默认只对园所管理员、平台运营可见；普通老师不显示金额明细。

### 导出与分享

- 导出要求读本存在且老师有权限；`pdf` 和 `print_layout` 要求至少有可展示页面。
- 如果仍有 `story_status != story_ready`，导出返回 `STATE_CONFLICT`。
- 如果图片未完成，允许文字版导出必须显式传入 `allow_text_only = true`，否则返回 `ILLUSTRATION_NOT_READY`。
- 分享默认 `private`；创建家庭分享链接使用 `family`，园内共享使用 `school`。
- `platform_public` 必须先进入 `platform_review`，审核通过后才能公开。
- 平台公开必须 `anonymize_child_name = true` 且 `anonymize_parent_info = true`。
- 共享库复制改编必须替换或清空强绑定原儿童/家长的角色和参考图。

## Auth And Permissions

### 角色

- `teacher`: 管理自己授权范围内的儿童、读本、导出和家庭分享。
- `school_admin`: 管理同园老师、班级、园内共享、成本聚合。
- `operator`: 管理模板、案例、平台审核、跨园公开内容。
- `parent_invited`: 仅能通过邀请 token 提交或更新一次家长资料。

### 权限矩阵

| 资源 | teacher | school_admin | operator | parent_invited |
|---|---|---|---|---|
| children | 自己班级或授权可读写 | 同园可读，按策略可写 | 仅运维排查 | 仅提交入口 |
| parents | 与自己儿童绑定可读 | 同园可读 | 仅运维排查 | 写自己的提交 |
| case_storybooks | published 可读 | published 可读 | 全量读写 | 不可见 |
| story_templates | 不可见 | 不可见 | 全量读写 | 不可见 |
| storybooks | 自己创建或授权可读写 | 同园共享可管理 | 公开审核可读 | 分享链接只读 |
| image_tasks | 自己读本相关可读写 | 同园统计可读 | 全量排查 | 不可见 |
| costs | 不可见或汇总 | 同园汇总 | 全量 | 不可见 |
| share_links | 自己读本可管理 | 同园可下架 | 平台公开可审核 | 链接只读 |

### 可见性规则

- 私有读本只对创建老师和具备管理权限的园所管理员可见。
- 园内共享读本对同一 `school_id` 的老师可见，但默认只允许复制，不允许编辑原件。
- 平台公开读本只展示脱敏后的案例视图。
- 家庭分享链接只能访问分享视图，不暴露编辑器字段、成本、内部任务快照。

## Error Model

统一错误包络：

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "请求参数校验失败",
    "details": [
      {
        "field": "name",
        "message": "姓名不能为空"
      }
    ],
    "request_id": "req_001"
  }
}
```

### 错误码

- `UNAUTHORIZED`: 未登录、token 失效或邀请 token 无效。
- `FORBIDDEN`: 已登录但没有访问权限。
- `NOT_FOUND`: 资源不存在，或用户无权访问时按安全策略隐藏为不存在。
- `VALIDATION_ERROR`: 字段格式、枚举、长度、必填校验失败。
- `STATE_CONFLICT`: 当前状态不允许该操作，例如故事未 ready 却创建图片任务。
- `IDEMPOTENCY_CONFLICT`: 同一个幂等键被不同请求体复用。
- `RESOURCE_CONFLICT`: 唯一约束冲突，例如重复 role_key 或重复 active 参考图。
- `REVIEW_REQUIRED`: 平台公开前必须审核。
- `RETRY_LIMIT_EXCEEDED`: 图片任务超过最大重试次数。
- `TASK_FAILED`: 异步任务失败且无法自动恢复。
- `EXPORT_FAILED`: 导出任务失败。
- `RATE_LIMITED`: 频率或额度限制。
- `INTERNAL_ERROR`: 未预期服务端错误。

### HTTP 状态建议

- 400: `VALIDATION_ERROR`
- 401: `UNAUTHORIZED`
- 403: `FORBIDDEN`
- 404: `NOT_FOUND`
- 409: `STATE_CONFLICT` / `RESOURCE_CONFLICT` / `IDEMPOTENCY_CONFLICT`
- 422: 业务规则不满足但请求格式正确
- 429: `RATE_LIMITED`
- 500: `INTERNAL_ERROR`

## State Transitions

### 读本生成

1. `draft` -> `story_generating`: 创建读本，开始故事生成。
2. `story_generating` -> `story_ready`: 页面正文、互动问题、角色清单、scene spec 写入完成。
3. `story_generating` -> `story_failed`: 故事生成失败，可重试故事生成或手动编辑。
4. `story_ready` -> `illustration_status = queued/running`: 创建图片任务。
5. 所有必需页面 ready -> `illustration_status = ready`。
6. 部分页失败 -> `illustration_status = partial_failed`，编辑器仍可处理成功页面。

### 图片任务

1. `queued` -> `running`
2. `running` -> `succeeded`
3. `running` -> `needs_review`
4. `running` -> `failed`
5. `queued/running` -> `cancelled`
6. `failed/needs_review` -> retry creates a new task with `retry_of_task_id`

### 分享

1. `private` -> `family`: 创建家庭分享链接。
2. `private/family` -> `school`: 老师明确园内共享。
3. `school/private` -> `platform_review`: 提交平台审核。
4. `platform_review` -> `platform_public`: 平台审核通过。
5. 任意共享状态 -> `private`: 创建者或管理员撤回。

## Cross-Layer Workflow Map

### 个性化读本主流程

| 用户动作 | UI 状态 | API | 数据写入 | 验证 |
|---|---|---|---|---|
| 老师选择孩子和案例 | 创建表单 | `POST /api/storybooks/generate` | `storybooks`、生成中状态 | child 可用，case published |
| 故事生成完成 | 编辑器可打开 | `GET /api/storybooks/:id` | `storybook_pages`、`storybook_roles` | story ready |
| 老师补充页面场景 | 单页编辑 | `PATCH /api/storybooks/:id/pages/:page_id` | `storybook_pages.scene_spec_json` | locked 策略，scene spec 合法 |
| 老师生成插图 | 图片任务面板 | `POST /api/storybook-pages/:page_id/image-tasks` | `image_generation_tasks` | scene ready，输入快照完整 |
| 老师选择候选图 | 候选图列表 | `POST /api/image-outputs/:id/select` | page current image | output 属于有效任务 |
| 老师导出 PDF | 导出任务 | `POST /api/storybooks/:id/exports` | `storybook_exports` | story ready，插图状态满足策略 |
| 老师分享给家长 | 分享弹窗 | `POST /api/storybooks/:id/share-links` | `storybook_share_links` | share scope 和脱敏规则 |

### 普通绘本派生定制绘本

| 用户动作 | API | 关键规则 |
|---|---|---|
| 选择普通绘本母本 | `POST /api/storybooks/:id/derive-custom` | 不覆盖母本，新建 custom storybook |
| 替换主角和家长角色 | `POST /api/storybooks/:id/replace-roles` | 更新 storybook_roles，按需标记图片 stale |
| 只重绘受影响页 | `POST /api/storybook-pages/:page_id/image-tasks` | 不默认整本重绘 |

## Review Notes

### Review 1: 数据库覆盖检查

- 已覆盖数据库设计中的 23 张表资源。
- `character_profiles` 不再被误认为只表示人物外观，接口补充了 `parent_character_profiles`、`prop_profiles` 和 `reference_images`。
- 关键道具多页出现通过 `prop_profiles` + `storybook_page_visual_subjects` 表达。
- 图片候选、审核和成本分别落到 `image_generation_outputs`、`image_review_events`、`generation_cost_logs`，避免塞进任务表或资产 metadata。

### Review 2: 业务流程检查

- 故事生成接口只保证故事和页面落库，不把图片生成作为前置。
- 图片生成入口要求 `story_status = story_ready` 和页面 `scene_spec_status = ready`。
- 普通绘本派生定制绘本不会覆盖母本。
- 角色替换优先复用已有内容，并只标记受影响页面，符合“不要默认连续重绘整本”的产品约束。

### Review 3: 校验、权限和错误检查

- 每个写模块都有必填、枚举、归属、状态和唯一性规则。
- 平台公开分享有审核和强制脱敏要求；家庭分享与园内共享分开。
- 图片重试区分自动可重试失败和需要人工调整的失败。
- 错误模型能支撑前端展示字段级错误、状态冲突、权限错误和异步任务失败。

### Review 4: 端点覆盖检查

- 端点矩阵中的每个接口都在逐端点验证矩阵中有对应规则。
- `image_assets` 已补充上传意图、资产登记和资产读取接口，避免照片、导出、二维码、生成图缺少统一资产入口。
- 内部表不强行暴露成 CRUD：`generation_cost_logs` 通过成本查询暴露，`image_review_events` 通过任务审核事件暴露，`image_generation_outputs` 通过任务详情、选择和审核暴露。
- 路由中的 `:page_id`、`:case_id`、`:template_id` 是 URL 参数命名；写入数据库时分别映射到 `storybook_pages.id`、`case_storybooks.id`、`story_templates.id`。

## Open Questions Or Versioning Risks

- 当前没有单独设计故事文本生成任务表，`POST /api/storybooks/generate` 通过 `storybooks.story_status` 暴露进度；如果后续需要详细追踪 LLM 成本，可新增 `story_generation_tasks`。
- 家长提交当前按邀请 token 设计；如果要做家长账号体系，需要新增 parent auth 和绑定流程。
- 平台公开审核目前复用 `submit-platform-review` 和分享状态；如果审核流变复杂，需要新增审核资源和审核事件表。
- 额度和计费当前只覆盖成本统计，没有订单、套餐和权益接口；商业化上线前需要单独设计 billing API。
