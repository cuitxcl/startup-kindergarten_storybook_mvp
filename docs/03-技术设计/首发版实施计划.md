# 幼儿园个性化内容平台首发版实施计划

## Build assumptions

- 当前目标是完成可推广的首发版，而不是轻量试用版
- 已有产品文档、图片链路和商业化方向已较完整，实施重点是收敛并打通
- 虽然目标是一步到位做全，但工程上仍需分阶段推进，避免多模块互相阻塞
- 园所、老师、家长、图片任务、共享治理都是首发范围的一部分

## Dependency chain

1. 先固定完整版数据边界和角色权限
2. 再完成核心迁移与基础枚举
3. 再搭建老师端、家长端和图片任务 API
4. 再实现老师主工作台和编辑器
5. 最后补齐共享、管理、额度和推广前验收

关键依赖：

- 家长提交页依赖 `parents + children + child_photos`
- 案例选择页和编辑页依赖 `storybooks + story_templates + character_profiles + reference_images + image_generation_tasks`
- 共享库依赖 `share_scope`、脱敏规则和来源关系
- 园所管理依赖组织、老师、班级和额度数据
- 在编码前必须先固定 `content_type` 和 `derivation_type` 枚举，避免前后端语义漂移

## Phase 1

目标：
- 搭起完整平台骨架，并优先打通老师生产链路

工作内容：

- 数据层
  - 创建 `schools`
  - 创建 `teachers`
  - 创建 `classrooms`
  - 创建 `parents`
  - 创建 `children`
  - 创建 `child_photos`
  - 创建 `story_templates`
  - 创建 `storybooks`
  - 创建 `storybook_pages`
  - 创建 `character_profiles`
  - 创建 `reference_images`
  - 创建 `image_generation_tasks`
  - 创建 `image_assets`
  - 创建 `generation_cost_logs`
  - 写入多内容线模板骨架和首批成品案例数据
  - 固定 `plain_storybook` / `custom_storybook` 等内容类型枚举
  - 固定 `original` / `from_plain_storybook` 等派生类型枚举

- 后端
  - 登录与当前园所接口
  - `GET /api/children`
  - `POST /api/children`
  - `POST /api/parent-intakes`
  - `GET /api/parent-intakes`
  - `GET /api/cases`
  - `GET /api/cases/:id`
  - `POST /api/characters`
  - `GET /api/characters/:id`
  - `POST /api/images/reference`
  - `POST /api/storybooks/generate`
  - `GET /api/storybooks/:id`
  - `PATCH /api/storybooks/:id/pages/:pageNumber`
  - `POST /api/images/pages`
  - `GET /api/image-tasks/:taskId`
  - `POST /api/storybooks/:id/export/pdf`

- 前端
  - 登录页
  - 工作台
  - 儿童列表和详情
  - 家长提交状态页
  - 成品案例中心
  - 生成配置页
  - 读本编辑页
  - 导出与分享基础页

- 验证
  - 跑通“家长提交 -> 教师生成 -> 编辑 -> 导出”

## Phase 2

目标：
- 补齐首发版推广需要的复用、共享和园所管理能力

工作内容：

- 增加内容中心和四条内容线聚合
- 增加共享库列表、共享设置和复制改编
- 增加园所管理、班级管理和老师账号管理
- 增加额度和成本概览
- 增加电子版、分享链接和二维码
- 增加整本图片生成批任务和单页重试

## Phase 3 or later

目标：
- 在首发版交付后补强商业化和运营效率

工作内容：

- 平台公开共享审核后台
- 家长增值购买链路
- 更细的统计报表
- 创作者激励和精选案例运营
- 自动化内容审核与脱敏建议

## Validation checkpoints

- 检查点 1：组织层
  - 园所、老师、班级和家长提交流程可用

- 检查点 2：生产层
  - 老师能基于完整儿童档案生成可编辑内容

- 检查点 3：图片层
  - 角色图、单页图和整本图任务可追踪、可重试

- 检查点 4：交付层
  - PDF、电子版、链接和二维码都可交付

- 检查点 5：共享层
  - 园内共享、脱敏和复制改编可用

- 检查点 6：管理层
  - 园所后台能查看额度、成本和老师使用情况

## Main implementation risks

- 如果不严格定义优先级，“一步做全”会演变为多模块同时半成品
- 如果图片任务稳定性和共享治理落后于老师主流程，推广时会直接暴露短板
- 如果管理后台太晚完成，园所采购价值无法被有效展示
- 如果首批成品案例不覆盖四条内容线，老师端会看起来像空壳
- 如果只有案例没有稳定底层骨架，后续维护和批量优化成本会迅速失控
