# 前端信息架构与页面流程

## 1. UX Summary

本前端蓝图用于验证产品总纲中的完整功能是否在界面上可理解、可执行、可扩展。

核心体验原则：

- 用户始终知道自己当前处于哪个空间：个人空间还是园所空间。
- 普通绘本和定制绘本共用一套生成与编辑体验，差异来自数据来源和空间权限。
- 绘本市场是“开始创作”的入口之一，不是独立交易系统。
- 园所管理视图服务成员、班级、内容和投稿，不抢老师的创作主流程。
- 链接访问者不需要理解完整系统，也能完成资料提交或绘本查看。
- 所有生成内容都可编辑；所有公开、分享、投稿动作都必须有确认步骤。

第一版前端不追求复杂数据大屏，而是围绕以下主路径验证产品结构：

```text
注册/登录
-> 进入当前空间
-> 创建或复制普通绘本
-> 生成/确认方案
-> 确认角色与道具
-> 编辑分页内容
-> 导出
```

```text
选择普通绘本
-> 选择孩子资料
-> 生成定制方案
-> 确认
-> 生成定制副本
-> 编辑
-> 导出/分享
```

```text
school_admin 邀请老师
-> 老师接受邀请
-> 进入园所空间
```

```text
园所普通绘本
-> 投稿市场
-> 平台审核
-> 市场上架
-> 其他空间复制使用
```

## 2. Users And Jobs

### 2.1 personal_owner

任务：

- 管理个人空间。
- 创建普通绘本。
- 管理自己的孩子资料。
- 基于普通绘本生成定制绘本。
- 从市场复制模板到个人空间。
- 导出 PDF。

主要担心：

- 不知道当前绘本归属哪里。
- 不知道定制绘本会使用哪些孩子信息。
- 从市场复制后是否会改到市场原模板。

### 2.2 school_teacher

任务：

- 在园所空间创建普通绘本。
- 使用班级儿童档案生成定制绘本。
- 创建家长资料提交链接。
- 确认家长提交资料。
- 从市场复制模板到园所空间。
- 导出或分享绘本给家长。

主要担心：

- 个人空间和园所空间混淆。
- 班级儿童档案是否授权可用。
- 分享给家长前是否会泄露敏感信息。

### 2.3 school_admin

任务：

- 管理园所基础信息。
- 邀请老师。
- 管理班级。
- 查看园所内容。
- 将优秀普通绘本投稿到市场。
- 查看基础用量。

主要担心：

- 成员是否已经接受邀请。
- 哪些内容可以被市场公开。
- 园所内容和老师草稿的边界。

### 2.4 link_visitor

任务：

- 通过资料提交链接填写孩子资料。
- 通过分享链接查看或下载绘本。

主要担心：

- 不知道资料会被谁看到。
- 不知道是否必须注册。
- 链接过期后如何处理。

### 2.5 platform_operator

任务：

- 管理平台模板。
- 审核园所投稿。
- 上架或下架市场内容。

主要担心：

- 投稿内容是否包含儿童隐私。
- 审核动作是否可追踪。

## 3. Global Information Architecture

### 3.1 顶层结构

登录后主应用分为五个一级区域：

| 一级区域 | 适用角色 | 作用 |
| --- | --- | --- |
| 工作台 | personal_owner、school_teacher、school_admin | 当前空间的首页和待办入口 |
| 绘本 | personal_owner、school_teacher、school_admin | 普通绘本、定制绘本、编辑和导出 |
| 孩子资料/儿童档案 | personal_owner、school_teacher、school_admin | 个人孩子资料或园所儿童档案 |
| 绘本市场 | 所有登录用户，可部分公开 | 浏览模板、复制到空间、园所投稿 |
| 管理 | school_admin、platform_operator | 园所管理或平台审核后台 |

### 3.2 空间差异

同一套导航在不同空间中显示不同文案和能力。

| 区域 | 个人空间 | 园所空间 |
| --- | --- | --- |
| 工作台 | 我的工作台 | 园所工作台 |
| 绘本 | 我的绘本 | 园所绘本 |
| 孩子资料 | 我的孩子 | 班级儿童 |
| 市场 | 绘本市场 | 绘本市场 |
| 管理 | 不显示或仅个人设置 | 园所管理 |

### 3.3 当前空间组件

所有登录后页面顶部必须显示：

- 当前空间名称。
- 空间类型：个人 / 园所。
- 当前角色：个人 / 老师 / 管理员。
- 空间切换入口。
- 进入个人空间的固定入口。

空间切换行为：

- 用户只有一个空间时，仅显示当前空间，不展示复杂切换菜单。
- 用户有多个空间时，点击空间名称打开空间列表。
- 切换空间后，列表、首页、创建入口全部切换数据来源。
- 如果用户在编辑流程中切换空间，先提示保存草稿或取消切换。

### 3.4 全局主操作

默认主操作根据当前空间和角色变化：

| 当前状态 | 主操作 |
| --- | --- |
| 个人空间 | 新建普通绘本 |
| 园所空间 teacher | 新建普通绘本 |
| 园所空间 admin | 邀请老师或新建普通绘本，按空态决定 |
| 绘本市场 | 复制到当前空间 |
| 绘本详情 | 编辑 / 生成定制版 / 导出 |

## 5. Primary Flow

### 5.1 注册后进入个人空间

```text
注册
-> 创建账号
-> 自动创建个人空间
-> 进入个人空间工作台
-> 看到“新建普通绘本”和“从市场开始”
```

成功反馈：

- 显示当前空间为“个人空间”。
- 引导用户创建普通绘本或浏览市场。

### 5.2 接受老师邀请

```text
打开邀请链接
-> 查看园所名称、邀请人、角色、班级范围
-> 登录或注册
-> 接受邀请
-> 成为 school_teacher
-> 进入园所空间工作台
```

异常：

- 链接过期：提示联系园所管理员重新邀请。
- 邀请已撤回：提示邀请不可用。
- 已经是该园所成员：直接进入园所空间。

### 5.3 普通绘本生成

```text
新建普通绘本
-> 填写绘本需求
-> 生成绘本方案
-> 确认/调整方案
-> 生成角色与关键道具
-> 确认/调整角色
-> 生成分页图文
-> 编辑分页
-> 生成插图或保留插图描述
-> 预览
-> 保存/导出
```

退出点：

- 任一步保存草稿后返回绘本列表。
- 方案不满意可重新生成。
- 角色不满意可替换单个角色。

### 5.4 定制绘本生成

```text
从普通绘本点击“生成定制版”
-> 选择孩子资料/儿童档案
-> 系统检查资料完整度
-> 选择快速定制或标准定制
-> 生成定制方案
-> 确认/调整方案
-> 生成独立定制副本
-> 编辑副本
-> 重绘需要变化的插图
-> 导出/分享
```

关键提示：

- 显示使用了哪些孩子信息。
- 显示会改写哪些页面、重绘哪些插图。
- 明确不会覆盖原普通绘本。

### 5.5 市场复制

```text
进入绘本市场
-> 搜索/筛选模板
-> 查看模板详情
-> 点击“复制到当前空间”
-> 确认目标空间
-> 创建当前空间普通绘本副本
-> 进入绘本详情或编辑页
```

### 5.6 园所投稿市场

```text
school_admin 打开园所绘本详情
-> 点击“投稿到市场”
-> 查看投稿预览和隐私检查
-> 确认提交
-> 状态变为 submitted
-> platform_operator 审核
-> approved/listed 后进入市场
```

### 5.7 家长资料提交

```text
teacher 创建资料提交链接
-> 家长打开链接
-> 查看资料用途和园所名称
-> 填写孩子资料
-> 提交
-> 老师看到待确认资料
-> 老师确认后写入儿童档案
```

## 6. Layout Sections

### 6.1 登录/注册页

目的：完成账号认证，不引入空间概念细节。

主区域：

- 品牌和一句话定位。
- 登录/注册表单。
- 邀请上下文提示，如果来自 `/invite/:token`。
- 错误提示区。

主操作：

- 登录。
- 注册。

状态：

- 登录中。
- 验证失败。
- 邀请 token 已失效。
- 注册成功后进入个人空间或继续接受邀请。

### 6.2 邀请接受页

目的：让用户明确接受后会加入哪个园所、获得什么角色。

主区域：

- 园所名称。
- 邀请人。
- 角色：老师。
- 可访问班级。
- 登录/注册状态。
- 接受邀请按钮。

主操作：

- 接受邀请。

次操作：

- 切换账号。
- 返回个人空间。

隐私与权限提示：

- 接受后只在该园所空间获得老师权限。
- 个人空间内容不会自动共享给园所。

### 6.3 应用框架

主区域：

- 顶部栏：当前空间、空间切换、用户菜单。
- 左侧导航：工作台、绘本、孩子资料/儿童档案、市场、管理。
- 主内容区。
- 全局生成/导出任务提示，可放在顶部或右下状态入口。

移动端：

- 左侧导航折叠为底部或抽屉导航。
- 当前空间必须仍在顶部第一屏可见。

### 6.4 工作台

目的：让用户知道当前空间可以做什么，以及下一步最重要动作。

个人空间布局：

- 顶部：个人空间名称、主操作“新建普通绘本”。
- 快捷入口：从空白创建、从市场开始、管理孩子资料。
- 最近绘本。
- 待处理事项：定制方案待确认、导出失败等。
- 空态：引导创建第一本普通绘本。

园所老师布局：

- 顶部：园所名称、角色老师、授权班级。
- 快捷入口：新建普通绘本、从市场开始、班级儿童、家长资料链接。
- 最近园所绘本。
- 待处理事项：家长资料待确认、定制绘本待编辑、导出失败。

园所管理员布局：

- 顶部：园所名称、成员/班级/用量摘要。
- 快捷入口：邀请老师、创建班级、查看园所内容、市场投稿。
- 待处理事项：邀请未接受、投稿审核结果、资料待确认。

## 7. List And Detail Design

### 7.1 绘本列表

Workflow summary：

- 用户查找自己或园所空间中的绘本，并进入编辑、定制、导出或投稿动作。

List structure：

- 默认使用卡片 + 可切换紧凑表格。
- 卡片适合展示封面、标题、类型、状态；表格适合管理大量园所内容。

Default fields：

| 字段 | 说明 |
| --- | --- |
| 标题 | 绘本名称 |
| 类型 | 普通绘本 / 定制绘本 |
| 状态 | 草稿、方案待确认、角色待确认、编辑中、可导出 |
| 来源 | 空白创建、市场模板、普通绘本派生 |
| 所属空间 | 当前空间内通常隐藏，跨空间视图显示 |
| 创建者 | 园所空间显示 |
| 更新时间 | 默认排序 |
| 可见性 | 私有、园所可见、投稿中、市场上架 |

Default sort：

- 更新时间倒序。
- 有待处理状态的内容可置顶。

Filters：

- 类型：普通绘本 / 定制绘本。
- 状态。
- 创建者，园所空间可用。
- 班级/孩子，定制绘本可用。
- 来源：市场复制 / 自建 / 派生。
- 可见性。

Search：

- 标题。
- 主题。
- 教学目标。

Navigation：

- 点击卡片进入详情/编辑。
- 返回列表时保留筛选、排序、滚动位置。

Bulk actions：

- 第一版不建议做批量删除。
- 园所管理员后续可批量归档。

Empty states：

- 个人空间无绘本：提供“新建普通绘本”和“逛绘本市场”。
- 园所空间无绘本：提供“新建普通绘本”“从市场复制模板”。
- 筛选无结果：提示清除筛选。

### 7.2 绘本详情/编辑页

Purpose：

- 承载普通绘本和定制绘本的查看、编辑、导出、定制派生和投稿动作。

Main sections：

- 顶部状态栏：标题、类型、状态、当前空间、保存状态。
- 操作区：编辑、生成定制版、导出、分享、投稿市场。
- 左侧页面列表：页码、状态、是否需重绘。
- 中间预览：当前页图文预览。
- 右侧编辑面板：标题、正文、插图描述、教师提示、角色/道具引用。
- 底部或侧栏：生成记录、导出记录、来源信息。

Primary actions：

- 草稿状态：继续编辑。
- 普通绘本可导出：导出 PDF。
- 普通绘本：生成定制版。
- 园所 admin 查看普通绘本：投稿市场。

Permission states：

- 无编辑权：只读预览，隐藏编辑控件。
- link visitor：仅展示分享允许的内容和下载入口。
- 市场模板详情不在此页直接编辑，必须先复制。

### 7.3 孩子资料/儿童档案列表

Workflow summary：

- personal_owner 管理个人孩子资料；school_teacher/school_admin 管理园所儿童档案。

List structure：

- 表格或紧凑卡片。
- 个人空间可以卡片为主；园所空间建议表格为主。

Default fields：

| 字段 | 个人空间 | 园所空间 |
| --- | --- | --- |
| 称呼/昵称 | 显示 | 显示 |
| 年龄段 | 显示 | 显示 |
| 兴趣/特点 | 显示摘要 | 显示摘要 |
| 班级 | 不显示 | 显示 |
| 档案完整度 | 显示 | 显示 |
| 最近定制绘本 | 显示 | 显示 |
| 更新时间 | 显示 | 显示 |

Filters：

- 园所空间：班级、档案完整度、最近更新。
- 个人空间：可不做复杂筛选。

Primary action：

- 个人空间：新增孩子资料。
- 园所空间 teacher：新增儿童档案或创建家长资料链接。
- 园所空间 admin：新增儿童档案或管理班级。

Detail page：

- 基础资料。
- 个性化元素。
- 定制绘本历史。
- 家长提交来源，园所空间显示。
- 编辑记录，后续。

Validation：

- 称呼/昵称必填。
- 年龄段必填。
- 至少一个兴趣或特点用于定制绘本。

### 7.4 市场列表

Workflow summary：

- 用户找到适合的模板，复制到当前空间，或基于模板发起定制。

List structure：

- 卡片网格。
- 每张卡展示标题、适用年龄、使用场景、页数、来源、是否支持定制。

Default sort：

- 平台精选优先。
- 最近上架。
- 后续可按使用次数。

Filters：

- 年龄段。
- 使用场景。
- 主题。
- 页数。
- 来源：平台模板 / 园所投稿。
- 是否支持定制。

Search：

- 标题、主题、教学目标。

Template detail：

- 标题和简介。
- 示例页。
- 页数结构。
- 角色需求。
- 适用年龄和场景。
- 来源和上架信息。
- 主操作：复制到当前空间。
- 次操作：基于此创建定制绘本，如果当前空间有孩子资料。

Copy confirmation：

- 明确目标空间。
- 复制后会创建独立副本。
- 编辑副本不会影响市场原模板。

### 7.5 园所成员列表

Workflow summary：

- school_admin 邀请和管理老师。

List structure：

- 表格。

Default fields：

- 姓名。
- 联系方式。
- 角色。
- 状态：active / invited / suspended。
- 授权班级。

## 8. Creation Workflows

### 8.1 普通绘本生成向导

Route：

- `/app/:workspaceId/storybooks/new`

Step 1：需求填写

- 主题。
- 年龄段。
- 页数。
- 使用场景。
- 故事风格。
- 画面风格，默认折叠。

Step 2：绘本方案确认

- 标题。
- 简介。
- 教学目标。
- 页数结构。
- 角色需求。
- 关键道具/场景。
- 画面基调。

Actions：

- 确认方案。
- 修改方案。
- 重新生成方案。
- 保存草稿。

Step 3：角色与道具确认

- 主角。
- 重复配角。
- 引导者。
- 关键道具。

Actions：

- 修改角色。
- 替换角色。
- 添加/删除非必要配角。
- 确认角色。

Step 4：分页图文编辑

- 页面列表。
- 当前页编辑。
- 插图描述。
- 教师提问/提示。

Actions：

- 保存。
- 重新生成单页。
- 生成插图。

Step 5：预览导出

- 整本预览。
- 导出 PDF。
- 返回编辑。

State indicators：

- 方案待确认。
- 角色待确认。
- 正文编辑中。
- 插图生成中。
- 可导出。

### 8.2 定制绘本生成向导

Route：

- `/app/:workspaceId/storybooks/:storybookId/customize`

Step 1：选择孩子

- 个人空间显示“我的孩子”。
- 园所空间显示“班级儿童”。
- 显示档案完整度。

Step 2：档案检查

- 必需信息缺失时阻止继续。
- 建议信息缺失时允许继续但提示个性化较弱。

Step 3：选择定制强度

- 快速定制。
- 标准定制。

Step 4：定制方案确认

- 定制标题。
- 主角处理。
- 个性化元素。
- 改写页面。
- 重绘插图。
- 隐私提示。
- 预计消耗。

Step 5：生成副本并编辑

- 创建独立定制绘本。
- 跳转到绘本编辑页。

Confirmation copy：

- “这会创建一份新的定制绘本，不会修改原普通绘本。”

### 8.3 市场投稿向导

Route：

- `/app/:workspaceId/storybooks/:storybookId/submit-market`

仅 `school_admin` 可进入。

Step 1：投稿预览

- 标题。
- 页数结构。
- 示例页。
- 来源绘本。

Step 2：隐私检查

- 确认不是定制绘本。
- 确认不含儿童照片。
- 确认不含具体儿童姓名、行为问题、家庭信息。
- 选择是否显示园所名称。

Step 3：提交审核

- 提交到平台审核。
- 显示投稿状态。

## 11. State Matrix

| 状态 | 通用处理 |
| --- | --- |
| Initial load | 显示骨架屏，顶部保留当前空间占位 |
| Loading refresh | 保留已有内容，显示局部刷新状态 |
| Empty state | 提供下一步动作，不显示空白页 |
| Validation error | 字段旁提示错误和修复方式 |
| Server error | 显示可重试动作，保留用户已输入内容 |
| Success feedback | 保存、复制、邀请、提交后显示明确成功状态 |
| Permission restriction | 说明当前角色无权操作，并提供返回或切换空间 |
| Stale data | 编辑冲突时提示刷新或保留本地修改，后续 |
| Link expired | 链接页显示过期原因和联系发起人提示 |

### 11.1 生成流程错误

- 方案生成失败：允许重试，保留需求输入。
- 角色生成失败：允许重试，保留已确认方案。
- 分页生成失败：允许重试，保留已确认角色。
- 插图生成失败：允许单页重试，不阻止文字版导出，具体策略待定。

### 11.2 市场复制错误

- 未登录：提示登录后复制。
- 无当前空间：要求选择空间。
- 无权限复制到目标空间：提示切换空间或联系管理员。
- 模板已下架：提示不可复制，返回市场列表。

### 11.3 邀请错误

- 邀请过期：提示联系园所管理员重新邀请。
- 邀请撤回：提示邀请不可用。
- 账号不匹配，后续可提醒当前账号与邀请联系方式不同。
- 已接受：提示进入园所空间。

## 13. Accessibility And Privacy Notes

Accessibility：

- 所有步骤导航可键盘操作。
- 生成状态不能只靠颜色区分。
- 表单错误需要文本说明。
- 长流程需要清晰的当前位置和返回方式。
- 市场卡片和绘本卡片的主操作要有可读标签。

Privacy：

- 当前空间必须始终可见。
- 孩子资料、儿童档案、定制绘本、分享链接和市场投稿都属于敏感或半敏感区域。
- 定制绘本投稿市场第一版禁止。
- 家长链接页必须展示资料用途和接收园所。
- 市场投稿必须有隐私确认。

## 14. Open UI Risks

- 空间切换如果不明显，用户可能把个人内容误建到园所空间，或反过来。
- school_admin 和 school_teacher 如果共用一个首页，主操作可能混乱；需要根据角色调整首页优先级。
- 普通绘本生成和定制绘本生成步骤都较长，需要保存草稿和明确进度。
- 市场复制到空间的动作如果没有确认，用户可能不知道副本归属。
- 孩子资料和儿童档案如果文案混用，会让个人用户和老师困惑。
- 园所投稿市场涉及隐私，如果只做一个“投稿”按钮风险很高，必须有预览和确认。
- 链接访问页面不能依赖完整 AppShell，否则家长会被复杂导航干扰。

## 15. Acceptance Criteria

- 登录用户在任意主页面都能看到当前空间和当前身份。
- 用户能在个人空间完成普通绘本创建流程，并导出。
- 用户能从普通绘本发起定制绘本流程，并明确看到定制会使用哪些孩子信息。
- school_admin 能在管理页邀请老师，并看到邀请状态。
- 老师接受邀请后能进入对应园所空间。
- 市场模板详情页能复制模板到当前空间，并明确不会修改市场原模板。
- 园所普通绘本投稿市场前必须经过投稿预览和隐私确认。
- link_visitor 可以在无登录情况下完成资料提交或绘本查看，不看到主应用导航。
- 无权限操作不会只隐藏按钮；直接访问路由时也有明确的无权限页面。
