# 儿童定制读本功能优先级

## 目标

明确功能开发顺序，避免首期范围膨胀，确保优先完成最小可用闭环。

## P0

这些功能缺一不可，没有就无法完成 MVP 交付。

### 儿童档案与主角替换

原因：
- 没有儿童档案，就无法实现“指定儿童做主角”
- 这是产品与普通故事生成工具的核心差异

### 成品案例库

原因：
- 老师更容易基于成品案例判断是否可用
- 没有案例库，老师需要自己脑补最终效果，转化会下降

### 一键生成个人定制读本

原因：
- 是核心转化动作
- 决定老师是否觉得工具省时间

### 教师文案编辑

原因：
- 老师必须能修正文案
- 如果只能自动生成，实际可用性会明显下降

### PDF 导出

原因：
- 读本必须能交付
- 没有导出能力，无法进入真实教学和家园沟通场景

## P1

这些功能会显著提升体验，但不是首期闭环必需。

### 插图风格切换

原因：
- 能提升个性化感受
- 但首期只用一种稳定风格也可运行

### 单页重绘

原因：
- 可以减少整本重新生成带来的编辑负担
- 但不是第一天必须具备

### 电子版分享

原因：
- 家园沟通很重要
- 但如果已有 PDF，首期仍可交付

### 班级管理

原因：
- 支撑重复使用和班级规模化
- 但初始试点阶段可先弱化

### 读本历史记录

原因：
- 提升复用性
- 不是首个闭环的决定性能力

## P2

这些功能适合在 MVP 通过后再扩展。

### 批量生成全班版本

原因：
- 有明显效率价值
- 但会增加案例、角色和质量控制复杂度

### 拼音标注

原因：
- 有阅读辅助价值
- 不是首期刚需

### 音频朗读

原因：
- 有助于家庭共读
- 实现复杂度高于首期收益

### 家长端查看

原因：
- 有长期价值
- 首期可先用分享链接替代

### 教学目标自动匹配

原因：
- 能提升智能化程度
- 但不影响基础读本生成

### 阅读数据统计

原因：
- 更偏后期运营和效果分析
- 不影响产品可用性

## 推荐开发顺序

### 阶段 1：最小闭环

- 儿童档案
- 成品案例库
- 生成配置
- 读本生成
- 文案编辑
- PDF 导出

### 阶段 2：可复用和可传播

- 插图风格切换
- 单页重绘
- 分享链接
- 二维码
- 读本历史

### 阶段 3：班级化和产品化

- 班级管理
- 批量生成
- 家长端能力
- 数据统计

## 砍需求原则

如果开发资源有限，优先保留以下 3 项：

1. 儿童档案与主角替换
2. 成品案例复用与教师编辑
3. PDF 导出

这 3 项已经足以完成“定制读本交付”。

## 判断是否进入下一优先级的标准

只有在下面条件成立时，才应继续做 P1 或 P2：

- 老师已完成真实使用
- 老师愿意重复生成
- 导出内容有真实传播或打印行为
- 用户反馈主要问题已不是“能不能用”，而是“够不够方便”
