# 幼儿园个性化内容平台首发版前端界面蓝图

## UI summary

前端界面需要同时支撑老师生产内容、家长提交资料、园所管理使用、共享内容复用和导出交付。虽然功能完整，但首层结构仍应围绕老师熟悉的任务组织，而不是按技术模块堆叠。

## Screens or routes

建议首发版使用以下核心路由：

- `/login`
  - 教师登录
- `/dashboard`
  - 工作台
- `/content`
  - 内容中心
- `/children`
  - 儿童档案列表
- `/children/:id`
  - 儿童详情与角色设定
- `/parent-intakes`
  - 家长提交状态管理
- `/cases`
  - 成品案例中心
- `/content/new`
  - 生成配置页
- `/storybooks/:id`
  - 读本编辑页
- `/exports/:id`
  - 导出与分享页
- `/shared-library`
  - 内容共享库
- `/admin/school`
  - 园所管理
- `/admin/credits`
  - 数据与额度

## Component responsibilities

- `DashboardPage`
  - 聚合内容入口、待办和最近内容
- `StorybookCard`
  - 展示成品内容摘要和状态
- `ChildList`
  - 展示儿童列表并处理选择行为
- `ChildForm`
  - 负责儿童基础信息创建与校验
- `ParentIntakeStatusTable`
  - 管理家长提交流程
- `CharacterProfilePanel`
  - 管理儿童与家长角色设定
- `CaseFilterBar`
  - 负责成品案例筛选条件管理
- `CaseCard`
  - 展示成品案例摘要和选择动作
- `ContentGenerationForm`
  - 收集完整生成配置并提交生成动作
- `RoleReplacementPanel`
  - 基于普通绘本母本替换主角或配角，派生定制绘本
- `StoryPageNavigator`
  - 负责页码切换和当前页状态提示
- `StoryPageEditor`
  - 负责单页文案编辑
- `ImageRegenerationPanel`
  - 发起单页重绘和显示任务状态
- `ExportButton`
  - 负责触发导出和显示导出中状态
- `ShareSettingsPanel`
  - 设置共享范围和脱敏选项
- `SharedLibraryCard`
  - 展示共享案例并支持复制改编
- `SchoolAdminPanel`
  - 管理老师、班级和园所配置
- `CreditUsagePanel`
  - 展示额度与成本

## User actions

- 老师从工作台选择内容线并新建内容
- 老师审核家长提交的儿童资料并补充教学信息
- 老师设置角色卡或发起参考图生成
- 老师筛选成品案例并完成生成配置
- 老师可先生成普通绘本，再从母本派生定制绘本
- 老师在编辑页修改文案、锁定页面、发起单页重绘
- 老师导出 PDF、电子版并配置分享
- 老师将内容设为园内共享或提交公开审核
- 其他老师从共享库复制改编
- 园所管理员查看额度与使用情况

## State matrix

### 初始加载

- 工作台、内容中心、共享库和管理后台都显示骨架状态
- 图片任务状态区在异步处理中显示队列与进度反馈

### Loading refresh

- 刷新列表时保留旧内容，并在局部区域显示加载提示
- 生成、重绘、导出和共享审核提交都采用局部 loading

### Empty state

- 没有儿童时，提示邀请家长提交或老师手动创建
- 没有最近内容时，提示从四条内容线发起第一个内容
- 没有共享内容时，提示先从园内优秀案例开始沉淀

### Validation error

- 儿童表单缺少姓名时，字段下方显示错误
- 家长资料未完成时，生成页提示缺失项
- 页面正文为空时，保存前提示具体页码
- 共享公开前若脱敏未完成，阻止提交

### Server error

- 列表加载失败时提供重试按钮
- 图片任务失败时展示失败原因和重试入口
- 生成失败时保留当前配置并允许再次提交
- 导出失败时显示错误 toast 和重试入口

### Success feedback

- 儿童创建成功后返回列表并高亮新记录
- 家长资料审核通过后更新状态标记
- 角色图生成成功后更新角色区状态
- 内容生成成功后自动跳转到编辑页
- 从普通绘本派生定制绘本成功后，自动进入新的定制版本编辑页
- 单页重绘成功后刷新当前页预览
- 导出成功后提示文件已生成
- 共享提交成功后提示当前共享范围

### Permission or auth restriction

- 未登录时跳转登录页
- 无访问权限的读本显示“无权查看该内容”
- 非管理员隐藏园所管理和额度模块

## Interaction notes

- 主按钮必须始终明确，避免同屏出现多个强竞争主动作。
- 老师端主路径始终是“选内容 -> 配置 -> 生成 -> 编辑 -> 导出”。
- 园所管理和共享治理应从老师生产界面中分层，不应混在同一个页面里。
- 异步图片生成和重绘必须有可见状态，否则老师会误判系统卡住。
- 共享提交前应展示脱敏结果预览，降低误操作风险。
- 如果当前内容来自普通绘本母本，界面应明确显示“派生自哪一本普通绘本”，避免老师误覆盖母本。

## Open UI risks

- 如果儿童详情同时承载家长资料、角色卡和历史内容而没有分区，页面会过载。
- 如果编辑页同时塞入文字、图片任务、共享设置和导出，会压垮主要编辑任务。
- 如果园所管理入口过于隐蔽，推广时会削弱 B 端采购价值感知。
- 如果共享库内容缺少脱敏标识和来源说明，老师会对复用边界产生疑虑。
