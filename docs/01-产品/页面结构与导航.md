# 页面结构与导航

## 目标

基于当前产品定位和信息架构，定义页面层面的结构与导航关系，明确：
- 一级导航和二级导航如何组织
- 核心页面有哪些
- 各页面之间如何跳转
- 首页如何体现“平台而不是单一绘本工具”

这份文档用于后续：
- 产品原型设计
- 页面清单梳理
- 前端路由设计

## 一、导航设计原则

### 原则 1：先按老师任务组织，再按系统能力组织

老师进入产品时，最关心的是：
- 我现在要做什么内容
- 我要给哪个孩子做
- 我怎么找到之前做过的内容

所以导航应优先围绕任务组织，而不是围绕技术模块组织。

### 原则 2：首层导航不要过多

首版建议控制在 5 个一级入口以内，避免老师进入系统后感到复杂。

### 原则 3：内容类型在首页和内容中心都应被看见

因为产品不能只被理解为“绘本工具”，所以四个内容场景必须在首页和内容中心都明确呈现。

## 二、一级导航建议

建议采用以下一级导航：

1. 工作台
2. 内容中心
3. 儿童档案
4. 成品案例中心
5. 导出与分享

如果后续做园所版增强，再扩展：

6. 园所管理
7. 数据与额度

## 三、一级导航说明

## 1. 工作台

### 作用

作为老师每天进入系统后的起点。

### 老师会在这里做什么

- 发起新内容
- 查看最近内容
- 回到常用案例
- 快速找到某个孩子

### 页面内容建议

- 顶部欢迎区
- 四个内容类型快捷入口
- 最近生成内容
- 常用案例
- 儿童档案快捷入口
- 班级概览或本周内容概览

## 2. 内容中心

### 作用

统一承载所有已生成和可新建的内容。

### 老师会在这里做什么

- 按内容类型查看内容
- 搜索某个孩子相关内容
- 继续编辑历史内容
- 复制已有内容给其他孩子

### 页面内容建议

- 内容类型切换标签
- 搜索框
- 筛选区
- 内容列表
- 新建内容按钮

## 3. 儿童档案

### 作用

管理所有孩子的个性化资料。

### 老师会在这里做什么

- 新增孩子
- 编辑孩子信息
- 上传照片
- 查看某个孩子已有内容

### 页面内容建议

- 儿童列表
- 搜索和筛选
- 新增儿童按钮
- 儿童详情卡

## 4. 成品案例中心

### 作用

统一管理所有成品案例，并按场景和主题组织。

### 老师会在这里做什么

- 找案例
- 预览案例
- 从案例开始新建内容
- 将普通绘本作为母本派生儿童定制绘本

### 页面内容建议

- 内容类型筛选
- 教学主题筛选
- 年龄段筛选
- 案例卡片列表

## 5. 导出与分享

### 作用

统一处理内容交付。

### 老师会在这里做什么

- 导出 PDF
- 生成电子版
- 获取分享链接
- 准备打印版本

### 页面内容建议

- 导出任务列表
- 分享记录
- 打印设置
- 常用导出格式入口

## 四、页面层级结构

建议页面结构分成三层：

### 第一层：一级模块页

- 工作台
- 内容中心
- 儿童档案
- 成品案例中心
- 导出与分享

### 第二层：列表或入口页

- 教学读本列表
- 情绪与规则故事列表
- 家园沟通内容列表
- 节日与成长纪念列表
- 儿童列表
- 案例列表

### 第三层：详情或编辑页

- 内容编辑页
- 儿童详情页
- 案例详情页
- 导出详情页

## 五、工作台页面结构

## 页面目标

让老师一进入系统就能快速开始，而不是面对复杂系统菜单。

## 页面模块建议

### 1. 顶部信息区

显示：
- 当前老师
- 当前班级
- 快捷搜索

### 2. 四个内容场景入口

以卡片形式直接展示：
- 个性化教学读本
- 情绪与规则引导故事
- 家园沟通内容
- 节日与成长纪念内容

每个卡片建议包含：
- 简短说明
- 最近使用数量
- 快速新建按钮

### 3. 最近生成

显示最近编辑或生成的内容，支持：
- 继续编辑
- 导出
- 复制

### 4. 常用案例

展示最近高频使用的案例。

### 5. 常用儿童入口

展示最近高频使用的孩子档案。

## 六、内容中心页面结构

## 页面目标

让老师把所有内容当成“工作内容”来管理，而不是散落在不同模块中。

## 建议结构

### 顶部操作区

- 新建内容按钮
- 搜索框
- 筛选按钮

### 内容类型导航

可采用 Tab 或分段导航：
- 全部内容
- 教学读本
- 情绪与规则故事
- 家园沟通内容
- 节日与成长纪念内容

### 列表区

每条内容卡片显示：
- 内容标题
- 内容类型
- 关联儿童
- 案例名称
- 母本来源
- 最近更新时间
- 状态
- 快捷操作

## 七、儿童档案页面结构

## 页面目标

让老师把“孩子”视为平台最重要的基础对象。

## 建议结构

### 左侧或顶部

- 搜索
- 班级筛选
- 标签筛选

### 主列表

儿童卡片显示：
- 姓名
- 昵称
- 年龄
- 标签
- 最近生成内容数
- 最近更新时间

### 点击进入儿童详情页

儿童详情页建议分为：
- 基本资料
- 照片与角色设定
- 已生成内容
- 快速新建内容

## 八、成品案例中心页面结构

## 页面目标

让老师快速找到适合当前教学任务的成品案例。

## 建议结构

### 顶部筛选区

- 内容类型
- 教学主题
- 年龄段
- 使用频率

### 案例列表区

案例卡片建议显示：
- 案例名称
- 适用内容类型
- 适用年龄
- 适用场景
- 页数或内容形态
- 预览入口
- 使用该案例按钮
- 生成普通绘本按钮

### 案例详情页

建议显示：
- 案例说明
- 教学目标
- 示例页
- 适用儿童场景
- 开始使用按钮
- 先生成普通绘本母本并派生定制绘本入口

## 九、内容编辑页结构

这是产品最核心的工作页。

## 页面目标

让老师快速审阅、编辑并交付内容。

## 建议三栏结构

### 左侧

- 页面缩略图或内容段落列表

### 中间

- 当前页预览
- 图文混合展示

### 右侧

- 文本编辑区
- 角色替换区
- 图片重绘区
- 页面设置区
- 锁定页面开关

### 顶部操作栏

- 保存
- 重新生成
- 导出
- 分享
- 复制给其他孩子
- 从普通绘本派生定制绘本

## 十、导出与分享页面结构

## 页面目标

让内容生成的结果更容易落地，而不是停在平台内部。

## 建议结构

### 导出入口区

- PDF 导出
- 电子版导出
- 打印版导出

### 分享入口区

- 家长分享链接
- 二维码
- 园所内部展示

### 历史记录区

- 最近导出
- 最近分享
- 导出状态

## 十一、关键跳转关系

建议核心跳转路径如下：

### 路径 1：从工作台新建内容

工作台  
-> 选择内容场景  
-> 案例选择  
-> 选择儿童  
-> 生成配置  
-> 内容编辑页  
-> 导出与分享

### 路径 2：从儿童档案发起内容

儿童档案  
-> 儿童详情  
-> 新建内容  
-> 选择内容类型  
-> 案例选择  
-> 生成配置  
-> 内容编辑页

### 路径 3：从案例发起内容

成品案例中心  
-> 案例详情  
-> 使用该案例  
-> 选择儿童  
-> 生成配置  
-> 内容编辑页

### 路径 4：复制历史内容

内容中心  
-> 历史内容详情  
-> 复制给其他孩子  
-> 替换儿童  
-> 优先复用现有成品并替换角色  
-> 内容编辑页

### 路径 5：从普通绘本派生定制绘本

内容中心  
-> 普通绘本详情  
-> 派生定制绘本  
-> 选择儿童和角色替换  
-> 生成新定制版本  
-> 内容编辑页

## 十二、导航层级建议

为了降低复杂度，建议：

### 左侧主导航

- 工作台
- 内容中心
- 儿童档案
- 成品案例中心
- 导出与分享

### 内容中心内部二级导航

- 全部内容
- 教学读本
- 情绪与规则故事
- 家园沟通内容
- 节日与成长纪念内容

### 右上角全局入口

- 搜索
- 通知或任务状态
- 账号和园所切换

## 十三、首版页面清单

首版建议至少包含以下页面：

1. 登录页
2. 工作台
3. 内容中心
4. 教学读本列表
5. 情绪与规则故事列表
6. 家园沟通内容列表
7. 节日与成长纪念内容列表
8. 儿童档案列表
9. 儿童详情页
10. 成品案例中心
11. 案例详情页
12. 生成配置页
13. 内容编辑页
14. 导出与分享页

## 十四、最终结论

页面结构应服务于一个核心目标：

让老师觉得这是一个围绕幼儿园日常任务组织的内容平台，而不是一个孤立的绘本生成器。

首版最合理的页面和导航逻辑是：
- 用工作台承接起点
- 用内容中心承接所有内容类型
- 用儿童档案承接个性化对象
- 用成品案例中心承接老师可直接复用的内容
- 用导出与分享承接交付

一句话概括：

`首页看起来像平台，编辑页用起来像工具，导出页体现真实交付。`
