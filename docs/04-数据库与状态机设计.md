# 数据库与状态机设计

## Domain Summary

数据模型围绕“账号 + 空间 + 绘本资产”展开。个人空间和园所空间共享绘本、儿童档案、市场模板等概念，但权限边界不同。普通绘本是可复用母本，定制绘本是从普通绘本和儿童档案派生出的独立副本。

## Entities

| Entity | 说明 |
|---|---|
| `users` | 登录账号 |
| `workspaces` | 个人空间、园所空间 |
| `workspace_members` | 用户在空间内的角色 |
| `classrooms` | 园所班级 |
| `children` | 儿童档案 |
| `parent_intakes` | 家长通过链接提交、待老师确认的儿童资料 |
| `storybooks` | 普通绘本、定制绘本 |
| `storybook_pages` | 绘本分页 |
| `storybook_roles` | 角色、老师形象、关键道具 |
| `marketplace_templates` | 市场模板 |
| `marketplace_submissions` | 园所投稿 |
| `share_links` | 分享链接 |
| `export_jobs` | 导出任务 |
| `generation_jobs` | 生成任务 |
| `audit_logs` | 权限、分享、投稿等关键操作审计 |

## Fields by Entity

- `users`：`id`、`display_name`、`email`、`password_hash`、`status`、`created_at`、`updated_at`。
- `workspaces`：`id`、`name`、`type`、`description`、`status`、`created_at`、`updated_at`。
- `workspace_members`：`id`、`workspace_id`、`user_id`、`role`、`status`、`classroom_ids`、`created_at`、`updated_at`。
- `classrooms`：`id`、`workspace_id`、`name`、`age_group`、`status`、`created_at`、`updated_at`。
- `children`：`id`、`workspace_id`、`classroom_id`、`nickname`、`age_group`、`interests`、`traits`、`focus`、`completeness`、`status`、`updated_at`。
- `parent_intakes`：`id`、`workspace_id`、`child_nickname`、`age_group`、`interests`、`status`、`confirmed_child_id`、`created_at`、`updated_at`。
- `storybooks`：`id`、`workspace_id`、`type`、`status`、`visibility`、`source`、`source_storybook_id`、`target_child_id`、`title`、`age_group`、`use_scene`、`teaching_goal`、`cover_tone`、`creator_id`、`created_at`、`updated_at`。
- `storybook_pages`：`id`、`storybook_id`、`page_number`、`title`、`body`、`illustration_prompt`、`status`。
- `storybook_roles`：`id`、`storybook_id`、`name`、`role_type`、`appearance`、`story_function`、`needs_consistency`。
- `marketplace_templates`：`id`、`source_type`、`source_workspace_id`、`source_storybook_id`、`title`、`summary`、`age_group`、`use_scene`、`page_count`、`supports_customization`、`tags`、`status`。
- `marketplace_submissions`：`id`、`workspace_id`、`source_storybook_id`、`title`、`submitted_by`、`status`、`privacy_confirmed`、`updated_at`。
- `share_links`：`id`、`storybook_id`、`token`、`scope`、`status`、`access_count`、`last_accessed_at`、`created_at`、`expires_at`。当前 API 创建时支持可选有效期；列表、公开查看和公开导出只接受 `active` 且未过期链接；公开查看成功时累计访问次数并记录最后访问时间。
- `export_jobs`：`id`、`storybook_id`、`status`、`file_url`、`last_error`、`created_at`、`finished_at`。
- `generation_jobs`：`id`、`workspace_id`、`storybook_id`、`job_type`、`status`、`input_json`、`output_json`、`attempt_count`、`last_error`、`next_run_at`、`locked_by`、`locked_at`、`created_at`、`finished_at`。当前 mock 执行器会在请求内走完 `queued -> running -> succeeded/failed`，后续真实 worker 继续复用这张表。
- `audit_logs`：`id`、`workspace_id`、`actor_user_id`、`action`、`resource_type`、`resource_id`、`metadata_json`、`created_at`。

## Relationships and Indexes

- `workspace_members.workspace_id + user_id` 唯一。
- `children.workspace_id + classroom_id` 支撑班级儿童列表。
- `parent_intakes.workspace_id + status + created_at` 支撑老师查看待确认家长提交资料。
- `storybooks.workspace_id + updated_at` 支撑绘本列表和工作台最近更新。
- `storybook_pages.storybook_id + page_number` 唯一。
- `marketplace_templates.status + source_type` 支撑市场筛选。
- `marketplace_submissions.workspace_id + status` 支撑投稿审核队列。
- `share_links.token` 唯一。

## State Machines

### Storybook

- `draft`：新建或复制后的草稿。
- `plan_pending`：故事方案待确认。
- `roles_pending`：角色和道具待确认。
- `editing`：分页文字和插图可编辑。
- `image_pending`：插图生成或重绘中。
- `exportable`：可导出、可分享、可派生定制。
- `submitted`：已作为市场投稿提交。
- `listed`：已进入市场。

允许路径：`draft -> plan_pending -> roles_pending -> editing -> image_pending -> exportable -> submitted -> listed`。编辑可从 `exportable` 回到 `editing`。

后端通用绘本更新接口需要校验状态迁移，不能允许前端任意 PATCH 到目标状态。当前允许：

- `draft -> plan_pending`
- `plan_pending -> roles_pending`
- `roles_pending -> editing`
- `editing -> image_pending`
- `editing -> exportable`
- `image_pending -> exportable`
- `image_pending -> editing`
- `exportable -> editing`
- `exportable -> submitted`
- `submitted -> listed`
- 同状态重复提交保持幂等

进入 `exportable` 前需要通过基础交付检查：绘本至少有一个分页、至少有一个角色或道具设定，且不存在 `generating` 状态的分页。`needs_regeneration` 暂作为交付提醒，不阻断老师先导出文字版或继续试点演示。

投稿隐私确认和平台审核可以在专用流程里驱动 `submitted/listed`，但仍应保持源绘本 `visibility` 同步为 `market_submission/market_listed`。

### MarketplaceSubmission

- `draft`：投稿草稿。
- `submitted`：已提交平台审核。
- `approved`：审核通过。
- `rejected`：审核驳回。
- `listed`：已上架市场。
- `unlisted`：已下架。

隐私确认是投稿进入 `submitted` 的 guard，不建议用 UI-only 状态代替。

### ParentIntake

- `submitted`：家长已提交，等待老师确认。
- `confirmed`：老师已确认并生成正式儿童档案，`confirmed_child_id` 指向新建的 `children` 记录。
- `rejected`：资料不采用或需要重新提交。

允许路径：`submitted -> confirmed`，后续可增加 `submitted -> rejected`。第一阶段确认后才写入正式儿童档案，避免 public 表单直接污染园所儿童数据。

### Jobs

- `queued`、`running`、`succeeded`、`failed`、`canceled`。
- 导出、图片生成、故事生成都可复用该生命周期。

## Migration Notes

一期 migration 建立账号、空间、儿童、绘本、市场、分享、导出和生成任务等核心表。后续补充 migration 用于收敛真实列表查询和流程约束所需的索引：

- `workspace_members(workspace_id, user_id)` 唯一，保证同一用户在同一空间只有一个成员身份。
- `classrooms(workspace_id, name)` 唯一，避免同一园所下出现重名班级。
- `children(workspace_id, classroom_id)` 支撑班级儿童列表和档案筛选。
- `parent_intakes(workspace_id, status, created_at desc)` 支撑老师处理家长资料待办。
- `storybooks(workspace_id, updated_at desc)` 支撑工作台和绘本列表最近更新。
- `storybook_pages(storybook_id, page_number)` 唯一，保证一本绘本内分页顺序稳定。
- `marketplace_templates(status, source_type)` 支撑市场筛选。
- `marketplace_templates(source_storybook_id)` partial index 支撑市场复制时克隆来源绘本内容。
- `marketplace_submissions(workspace_id, status)` 支撑园所投稿队列。
- `generation_jobs(workspace_id, status, created_at desc)` 支撑生成任务进度查询。
- `generation_jobs(status, next_run_at, created_at)` partial index 支撑失败任务按可重试时间扫描。
- `generation_jobs(locked_at)` partial index 支撑运营侧查找卡住的 running 任务。
- `export_jobs(storybook_id, status)` 支撑绘本导出任务查询。
- `export_jobs(storybook_id, created_at desc)` 支撑绘本详情最近导出分页。
- `share_links(storybook_id, status, created_at desc)` partial index 支撑绘本详情当前有效分享链接分页。

当前后端已进入 PostgreSQL + SeaORM repository 读写阶段，仍保留 mock 生成内容和 mock 导出结果；生成和导出已按任务语义先入队再由 mock 执行器完成。后续真实 AI、对象存储和任务队列接入时应继续保持 API 契约不变，优先参考 `docs/05-异步任务与生成队列设计.md`。

## Audit Logs

`audit_logs` 用于记录高影响、权限敏感或隐私相关的关键动作。审计记录只追加，不参与业务状态判断；业务状态仍以对应资源表为准。

第一阶段已落地的审计动作：

- `child.created`：登录用户创建儿童档案，用于后续定制绘本。
- `child.updated`：登录用户更新儿童档案，影响后续定制绘本使用的个性化元素。
- `storybook.created`：登录用户创建普通绘本，开始沉淀内容母本。
- `storybook.updated`：登录用户更新绘本元信息、状态或共享范围。
- `storybook.page_updated`：登录用户编辑绘本分页文字、插图描述或分页状态。
- `storybook.role_updated`：登录用户编辑绘本角色或道具设定。
- `storybook.custom_derived`：登录用户基于普通绘本和儿童档案生成定制绘本副本。
- `parent_intake.confirmed`：老师确认家长提交资料并生成正式儿童档案。
- `marketplace_template.copied`：登录用户从绘本市场复制模板到当前空间，生成可编辑普通绘本。
- `generation_job.created`：登录用户创建故事、角色、分页、插图或定制方案生成任务；审计只记录任务类型和状态摘要，不记录完整 prompt/input。
- `generation_job.retried`：登录用户重试失败的生成任务；审计记录任务类型、重试后状态和尝试次数。
- `generation_job.canceled`：登录用户取消排队中或失败待重试的生成任务；审计记录任务类型和取消后状态，不记录完整 prompt/input。
- `workspace_member.invited`：园所管理员邀请老师。
- `classroom.created`：园所管理员创建班级。
- `storybook.export_created`：登录用户为绘本创建导出任务。
- `storybook.share_link_created`：登录用户为绘本创建分享链接。
- `storybook.share_link_revoked`：登录用户撤回绘本分享链接，撤回后公开访问不可用。
- `share_link.public_export_created`：链接访问者通过分享链接创建导出任务，`actor_user_id` 为空。
- `marketplace_submission.created`：园所管理员创建市场投稿草稿。
- `marketplace_submission.privacy_confirmed`：园所管理员完成投稿隐私确认。
- `marketplace_submission.approved`：平台侧审核通过并上架市场模板。

审计字段口径：

- `workspace_id`：能归属到空间的操作必须填写；公开分享导出和平台审核可为空或后续补平台 workspace。
- `actor_user_id`：登录态操作填写当前用户；公开链接访问者为空。
- `resource_type/resource_id`：指向被操作的主要资源，例如 `share_link`、`export_job`、`marketplace_submission`。
- `metadata_json`：只记录非敏感摘要，例如状态、模板标题、班级名、授权班级；不记录密码、完整 token、儿童敏感描述全文。
