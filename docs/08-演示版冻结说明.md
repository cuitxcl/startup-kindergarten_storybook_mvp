# Kindleaf 本地演示版冻结说明

## 1. 冻结结论

当前项目已经达到“本地可演示闭环”标准，可以先冻结为演示版。

冻结口径不是生产上线，而是证明以下主路径能在本地真实 API + PostgreSQL 环境中跑通：

- 老师登录系统。
- 进入个人空间或园所空间。
- 查看工作台。
- 创建、编辑普通绘本。
- 维护儿童档案。
- 基于普通绘本生成单个儿童的定制绘本。
- 生成插图文件：演示环境默认 mock provider；真实生图模型已统一为字节跳动 Seedream，并具备 fake provider、真实 Seedream 试运行脚本和 DeepSeek+Seedream 组合验收脚本。
- 导出本地基础 PDF 文件。
- 创建和撤回分享链接。
- 园所投稿、隐私确认、平台审核上架。
- 从绘本市场复制模板回当前空间。
- 家长资料链接创建、公开提交、确认入档。
- 园所生成队列恢复。

最近一次完整验收命令：

```sh
./scripts/check-smart.sh demo
```

演示交接前也可运行 `CHECK_SMART_RUN_FULL=true ./scripts/check-smart.sh auto`、`./scripts/check-smart.sh full` 或 `./scripts/smoke-all.sh`；`demo` 和 `full` 都会固定跑完整 API + UI smoke，`demo` 作为语义更清楚的演示交接入口。

最近一次验收结果：

- fast check 通过。
- 前端 API build 通过。
- 后端测试通过：2026-07-21 最近一次快速验证为 116 个测试全部通过。
- API smoke 通过。
- Provider smoke 通过。
- 生成预算 smoke 通过。
- UI smoke 通过。
- 最终输出：`== smart check ok ==`；其中完整 smoke 输出：`== full smoke ok ==`。

最新一次完整验收还覆盖了生成成本账本、家长资料链接按班级批量停用、登录态 PDF 下载、公开分享 PDF 下载、插图文件权限下载、市场投稿审核、复制复用和园所管理页取消排队生成任务：生成任务完成后，平台运营端可通过 `/api/operator/generation-costs` 查到对应成本记录，普通用户访问该接口会被拒绝；管理员取消生成任务后，生成队列显示“已取消”，审计日志显示“取消生成任务”。生成成本专项验收还覆盖了预算预警、预算超限和超限后的生成任务拦截。

## 2. 演示环境启动

启动 PostgreSQL：

```sh
docker compose up -d postgres
```

运行 migration：

```sh
cd server
DATABASE_URL=postgres://postgres:postgres@127.0.0.1:55432/kindleaf_development \
  cargo run -p migration -- up
```

写入演示数据：

```sh
cd server
DATABASE_URL=postgres://postgres:postgres@127.0.0.1:55432/kindleaf_development \
  cargo run --features db -- db seed
```

启动后端：

```sh
cd server
DATABASE_URL=postgres://postgres:postgres@127.0.0.1:55432/kindleaf_development \
KINDLEAF_DEMO_SEED=1 \
KINDLEAF_GENERATION_PROVIDER=mock \
KINDLEAF_COST_BUDGET_WARNING_PERCENT=80 \
  cargo run --features db -- start
```

启动前端：

```sh
cd frontend
npm run dev:api
```

## 4. 推荐演示路径

### 4.1 园所管理员主线

1. 登录 `lin@example.com / demo`。
2. 进入 `星星幼儿园`。
3. 打开园所工作台，查看今日重点和绘本生产入口。
4. 创建普通绘本。
5. 在绘本详情中编辑文字、角色和页面。
6. 生成插图。
7. 标记可交付。
8. 导出 PDF。
9. 创建分享链接。
10. 进入园所管理，创建老师邀请、管理班级、查看投稿。
11. 将普通绘本投稿到市场。
12. 在平台运营后台审核上架。
13. 回到市场复制模板到当前空间。

### 4.2 儿童定制主线

1. 进入儿童档案页。
2. 新增儿童档案，填写称呼、年龄段、兴趣、特质和关注点。
3. 从普通绘本进入定制绘本流程。
4. 选择儿童。
5. 生成定制方案。
6. 生成定制副本。
7. 在定制绘本详情中标记可交付、导出或分享。

### 4.3 家长资料链接主线

1. 园所管理员进入儿童档案页。
2. 选择班级范围。
3. 生成家长资料链接。
4. 打开公开链接，提交孩子资料。
5. 回到儿童档案页，在待确认队列确认入档。
6. 确认后的儿童档案继承来源班级。
7. 可撤回链接或批量停用当前班级可填写链接。

### 4.4 园所老师边界

1. 切换到老师空间。
2. 老师只能看到授权班级内的儿童档案。
3. 老师不能进入园所管理页。
4. 老师查询生成任务时，原始输入会脱敏。

## 5. 已知限制

这些限制不阻碍本地演示，但阻碍真实试点或上线：

- 文本生成默认仍使用 mock provider。
- DeepSeek 文本 provider 已有 fake smoke、真实试运行脚本和 HTTP 边界，但真实生产密钥、配额、降级策略还未收口。
- 图片生成已统一到字节跳动 Seedream，已有 mock PNG、fake Seedream smoke、真实 Seedream 试运行脚本和 DeepSeek+Seedream 组合验收脚本；Seedream 输出写入前会拒绝非 PNG 内容。图片文件读写已抽出可配置 storage service，但图片审核、对象存储和生产文件生命周期还未生产化。
- PDF 导出已有本地基础 PDF 文件和权限下载，PDF 文件读写已抽出可配置 storage service；但专业排版模板、字体嵌入、插图排版和对象存储还未生产化。
- 邮件/短信邀请尚未接入，当前邀请链接仍通过页面展示和复制。
- 平台审核、隐私脱敏和内容安全还是基础版本。
- 商业化额度、套餐、账单和支付未实现。
- 当前演示依赖本地 PostgreSQL、Docker、Chrome headless 和本地端口。

## 7. 下一阶段边界

演示版冻结后，不建议继续扩大 mock 功能范围。

下一阶段应集中做真实生成和试点准备：

1. 真实文本生成：故事方案、角色设定、分页图文、定制方案。
2. 真实插图生成：图片 provider、失败重试、图片审核、对象存储。
3. PDF 排版升级：专业排版模板、字体嵌入、插图排版、对象存储和文件生命周期。
4. 邮件邀请：老师邀请和通知。
5. 试点安全：儿童资料隐私、投稿脱敏、操作审计、备份恢复。

判断标准也应从“页面能不能点”切换为：

- 生成结果是否可用。
- 数据是否可信。
- 异常是否可恢复。
- 权限是否不会越界。
- 运营是否能追踪和处置问题。

真实生成接入细节见 `docs/09-真实生成能力接入说明.md`。
