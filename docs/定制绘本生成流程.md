# 定制绘本生成流程

## 定义

定制绘本是基于一份已完成的普通绘本，为某个孩子生成的专属版本。

第一版的定制绘本不从零生成，而是从普通绘本派生：

```text
普通绘本
-> 选择孩子
-> 生成定制方案
-> 确认后生成独立定制副本
```

普通绘本提供稳定的故事结构、教学目标、角色关系和页面节奏；定制绘本只在必要位置加入孩子相关信息。

## 核心原则

- 定制绘本优先从普通绘本派生，不从零创作。
- 原普通绘本永远不被覆盖。
- 老师确认的是“这本书会怎样变成某个孩子的版本”，不是底层角色映射规则。
- 系统先生成可审核的定制方案，再生成完整定制副本。
- 儿童档案不足时要提示补充，避免生成空泛内容。
- 个性化强度要可控，方便后续扩展到全班批量定制。
- 隐私提示必须进入流程，不能藏在设置里。

## 推荐流程

```text
1. 选择普通绘本
2. 选择孩子
3. 检查儿童档案
4. 选择定制强度
5. 生成定制方案
6. 老师确认/调整定制方案
7. 生成独立定制绘本副本
8. 老师编辑副本
9. 生成或重绘需要变化的插图
10. 预览、导出 PDF 或分享给家长
```

## 步骤说明

### 1. 选择普通绘本

老师从已有普通绘本中选择一本作为来源。

适合作为来源的普通绘本应满足：

- 故事结构已确认
- 分页内容完整
- 角色设定清楚
- 教学目标明确
- 可直接预览或导出

定制绘本的效率来自复用普通绘本，所以第一版不建议从空白主题直接生成定制绘本。

### 2. 选择孩子

老师选择一个孩子作为定制对象。

孩子资料可能包含：

- 姓名或昵称
- 年龄段
- 班级
- 兴趣标签
- 性格特点
- 老师观察
- 教学关注点
- 喜欢的颜色、玩具或活动
- 外观或角色偏好

第一版不强制使用真实照片。没有照片时，可以生成“孩子化角色描述”。

### 3. 检查儿童档案

系统检查孩子资料是否足够支撑定制。

必需信息：

| 字段 | 用途 |
| --- | --- |
| 孩子称呼/昵称 | 用于标题、正文和主角称呼 |
| 年龄段或班级年龄 | 控制语言难度和故事表达 |
| 至少一个个性化元素 | 用于让故事不只是名字替换，例如兴趣、喜欢的玩具、近期关注点 |

建议补充信息：

| 字段 | 用途 |
| --- | --- |
| 性格特点 | 调整主角行为和情绪表达 |
| 老师观察 | 让故事更贴近真实场景 |
| 教学关注点 | 决定哪些页面需要改写 |
| 喜欢的颜色 | 可用于服装或道具 |
| 外观/角色偏好 | 可用于插图和角色设定 |
| 家庭共读寄语 | 可用于结尾页或分享页 |

规则：

- 缺少必需信息时，阻止生成并提示老师补充。
- 缺少建议信息时，允许继续，但提示“个性化程度会较弱”。

### 4. 选择定制强度

第一版只保留两个强度，避免流程复杂化。

| 模式 | 说明 | 适合场景 |
| --- | --- | --- |
| 快速定制 | 替换称呼、标题、关键道具，尽量保留原文和原图 | 批量生成、低成本交付 |
| 标准定制 | 改写关键页面，并重绘关键插图 | 单个孩子常规定制、家园沟通 |

暂不做“深度定制”。深度定制会接近重新生成一本书，成本和质量都更难控制。

### 5. 生成定制方案

系统根据普通绘本、孩子资料和定制强度，生成老师可审核的定制方案。

定制方案不直接展示底层角色映射，而是展示老师关心的变化。

方案应包含：

| 内容 | 示例 |
| --- | --- |
| 定制标题 | 《乐乐学会一起玩》 |
| 主角处理 | 原主角“小兔米米”会变成“乐乐” |
| 保留角色 | 小熊朋友、鹿老师保留 |
| 个性化元素 | 加入乐乐喜欢的积木车、蓝色书包 |
| 改写页面 | 封面、第 1 页、第 4 页、结尾页 |
| 重绘插图 | 封面、第 1 页、第 4 页、结尾页 |
| 隐私提示 | 将使用孩子昵称和兴趣，不使用真实照片 |
| 预计消耗 | 预计生成 4 页新插图 |

定制方案需要明确：

- 会改什么
- 会保留什么
- 哪些页面需要重新生成文字
- 哪些页面需要重新生成插图
- 使用了哪些儿童信息

### 6. 老师确认/调整定制方案

老师可以调整：

- 定制标题
- 孩子称呼
- 使用哪些个性化元素
- 哪些页面需要改写
- 哪些插图需要重绘
- 是否保留原配角
- 是否加入家庭共读寄语

这一阶段的目标是确认“定制方向”，不是逐字编辑完整绘本。

老师确认定制方案后，系统再生成独立定制副本。

### 7. 生成独立定制绘本副本

系统生成新的定制绘本，不覆盖原普通绘本。

定制副本应记录：

- 来源普通绘本
- 定制对象
- 定制强度
- 使用的儿童档案字段
- 页面改写范围
- 插图重绘范围
- 创建老师
- 创建时间

生成内容包括：

- 定制标题
- 定制后的分页正文
- 定制后的插图描述
- 保留页面
- 改写页面
- 需要重绘的图片任务

### 8. 老师编辑副本

老师可以继续逐页编辑：

- 页面标题
- 页面正文
- 插图描述
- 教师提问
- 教师提示
- 家庭共读寄语

普通绘本和定制绘本是两个独立内容，修改定制副本不会影响普通绘本。

### 9. 生成或重绘需要变化的插图

第一版不需要重绘所有页面。

推荐策略：

- 封面需要个性化重绘。
- 孩子明显出现的页面需要重绘。
- 关键道具替换的页面需要重绘。
- 背景页、老师引导页、无明显主角变化的页面可复用普通绘本插图。

系统需要告诉老师：

- 哪些图会复用
- 哪些图会重绘
- 哪些图还在生成
- 哪些图生成失败，可重试

### 10. 预览、导出 PDF 或分享给家长

老师最终可以：

- 预览定制绘本
- 导出 PDF
- 生成家长分享链接
- 复制生成另一个孩子版本
- 返回编辑

第一版可以先支持 PDF 导出，家长分享链接可作为增强项。

## 定制方案示例

```text
来源普通绘本：《一起玩小汽车》
定制对象：乐乐
定制模式：标准定制

定制标题：《乐乐学会一起玩》

会改：
- 标题中的主角名
- 封面主角
- 第 1 页开场，把“小兔米米”改为“乐乐”
- 第 4 页练习轮流的道具，把“红色小汽车”替换成“彩色积木车”
- 结尾页加入给乐乐的鼓励语

会保留：
- 故事主线
- 页数结构
- 小熊朋友
- 鹿老师
- 教学目标

会重绘插图：
- 封面
- 第 1 页
- 第 4 页
- 第 6 页

隐私提示：
- 使用孩子昵称“乐乐”
- 使用兴趣信息“喜欢积木车”
- 不使用真实照片
```

## 状态设计

| 状态 | 含义 |
| --- | --- |
| 选择来源中 | 老师还没有选择普通绘本 |
| 选择孩子中 | 老师还没有选择定制对象 |
| 档案待补充 | 必需儿童信息不足 |
| 定制方案待生成 | 已选择来源和孩子，可生成方案 |
| 定制方案待确认 | 系统已生成方案，老师未确认 |
| 定制副本生成中 | 正在生成独立定制绘本 |
| 定制编辑中 | 老师可编辑定制副本 |
| 插图生成中 | 正在生成或重绘定制插图 |
| 可导出 | 定制绘本可预览、导出或分享 |

## 第一版范围

必须做：

- 从普通绘本发起定制
- 选择单个孩子
- 儿童档案必需字段检查
- 快速定制/标准定制两种强度
- 生成定制方案
- 老师确认/调整定制方案
- 生成独立定制副本
- 逐页编辑定制副本
- PDF 导出

可以延后：

- 全班批量定制
- 深度定制
- 家长端账号
- 家长在线确认
- 支付
- 平台公开共享
- 全量图片重绘
- 使用真实儿童照片生成角色参考图

## 关键判断

定制绘本流程应定为：

```text
选择普通绘本和孩子，先生成老师看得懂的定制方案，确认后再生成独立定制副本。
```

这比直接暴露“角色映射”更符合老师的使用习惯，也为后续批量定制保留了可控的生成策略。
